总体架构概览

🟦 直接复用 OSS 🟩 封装适配 🟥 自研核心 🟪 专用 Agent 角色

AgentFlow 是一个将可视化工作流编排与AI Agent 自动执行结合的平台。用户通过画布节点定义工作流 DAG，系统自动编译为 Agent 可执行的提示词，通过 AgentRunner 引擎调用 Multi-Provider LLM 执行，形成完整的闭环反馈系统。

🎯 核心公式 画布节点 + 6 Profile 模板 → Compiler Agent → PromptTask[] → AgentRunner (Multi-Provider) → Envelope 结果 → 画布反馈

🎯 核心理念

用户说需求 → 自动编排 → 用户确认 → 自动执行 → 结果反馈。每个节点 → 完整自然语言提示词 → LLM 执行。

🔁 闭环反馈

下游测试失败 → 回退上游重生成 → 控制理论负反馈。Lyapunov 稳定：每次迭代 dV/dt < 0。

⚡ 执行单元

每个 Agent = AgentRunner (Python) + Multi-Provider LLM 调用。支持 OpenAI / GLM 等多模型后端。

📐 数据契约

三份核心 JSON Schema：WorkflowJSON / PromptTask / Envelope。每个环节 schema 门控验证。

画布 UI — CSS Grid 实现

7.1 概述

画布 UI 是 AgentFlow 面向用户的核心交互界面。当前实现为原生 CSS Grid 单页应用 (canvas-demo.html v6)，不依赖 React 或 React Flow。

🎨 布局引擎

CSS Grid 四列：step-number(30px) | agent-card(220px) | gap(32px) | output-card(200px)

🌙 暗色主题

CSS 变量体系：--bg, --surface, --border, --text, --accent, --green 等

📱 响应式

三面板设计：190px 侧栏 + flex 画布 + 310px 配置面板

7.2 CSS Grid 布局

/* CSS Grid 核心 — 横平竖直 中心对齐 */ .canvas-grid-inner { display: grid; grid-template-columns: 30px 220px 32px 200px; grid-auto-rows: 76px; align-items: center; /* 垂直中心 */ justify-items: center; /* 水平中心 */ gap: 4px 0; }
列宽度内容说明
step-number30px01, 02...步骤编号，右对齐
agent-card220pxAgent 卡片图标 + 名称 + Profile + 状态徽章
gap/arrow32px→水平箭头表示数据流方向
output-card200px输出卡片显示 Agent 执行结果

7.5 与原始设计（React Flow）的差异

维度原始设计 (Spec)实际实现 (v6)
画布框架React Flow (xyflow)原生 CSS Grid
节点拖拽侧边栏拖入画布硬编码 palette，无拖拽
连线功能Port-to-Port 拖线 + 类型检查箭头图标 →，无实际连线
工作流验证循环检测 + 类型匹配无
导出 WfJSONCtrl+S / 右键导出后端序列化
自动布局dagre 自动布局CSS Grid 固定行列
撤销/重做快照栈（50 步）无
状态推送WebSocketREST + 前端模拟
执行引擎Docker + Claude CodeAgentRunner (Multi-Provider)
设计决策：CSS Grid 替代 React Flow 是为了降低 MVP 复杂度，同时在视觉上实现「横平竖直」的清晰布局。未来 Phase 3 可迁移回 React Flow 以获得完整拖拽/连线能力。
