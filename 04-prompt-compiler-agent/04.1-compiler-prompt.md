# 04.1 — Compiler 编译流程

## 1. 概述

Compiler 是 AgentFlow 的数据转换层（`prompt_compiler.py`），将 WorkflowJSON 编译为 PromptTask[]。它不是一个 AI Agent，而是一个确定性的转换引擎——不调用 LLM，不做推理，只做编译。

```
WorkflowJSON
    │
    │  nodes[].profile → 决定加载哪个 YAML 模板
    │  edges[]         → 决定依赖关系和上游上下文
    │
    ▼
┌─────────────────────────────────────────────┐
│               PromptCompiler                  │
│                                               │
│  步骤 ①: 解析 WorkflowJSON                    │
│  步骤 ②: Kahn 拓扑排序 → 深度分组             │
│  步骤 ③: 加载 YAML 模板                       │
│  步骤 ④: 模板变量渲染                         │
│  步骤 ⑤: 构建 upstream_context                │
│  步骤 ⑥: 组装 PromptTask[]                    │
│                                               │
└─────────────────────────────────────────────┘
    │
    ▼
PromptTask[]  →  Orchestrator 调度执行
```

## 2. 步骤详解

### 步骤 ①：解析 WorkflowJSON

**输入：** 前端传来的 `nodes[]` + `edges[]` + `requirement`

```python
def handle_execute(self):
    body = json.loads(self.rfile.read(length))
    nodes = body.get("nodes", [])
    edges = body.get("edges", [])
    requirement = body.get("requirement", "")

    # 构建 WorkflowJSON 对象
    wf = WorkflowJSON.from_api_request(nodes, requirement, edges)
    errs = validate_workflow(wf)  # 校验节点和边的完整性
```

**校验规则：**
- `nodes` 非空
- 每条 edge 的 `source` 和 `target` 必须存在于 `nodes`
- 不允许自环（`source == target`）
- 每个节点的 `profile` 必须是 `analysis/design/dev/test/doc/deploy` 之一

### 步骤 ②：Kahn 拓扑排序 → 并行分组

使用 Kahn 算法对有向无环图（DAG）进行拓扑排序，并计算每个节点的深度层级。

```python
def topological_sort(nodes, edges) -> tuple[list[NodeDef], dict[str, int]]:
    # 1. 建立邻接表和入度表
    in_degree = {n.id: 0 for n in nodes}
    adjacency = {n.id: [] for n in nodes}
    for e in edges:
        adjacency[e.source].append(e.target)
        in_degree[e.target] += 1

    # 2. BFS 拓扑排序——入度为 0 的节点优先
    queue = [nid for nid, deg in in_degree.items() if deg == 0]
    depth = {nid: 0 for nid in queue}

    while queue:
        nid = queue.pop(0)
        sorted_ids.append(nid)
        for downstream in adjacency.get(nid, []):
            in_degree[downstream] -= 1
            if in_degree[downstream] == 0:
                depth[downstream] = max(depth.get(downstream, 0), depth[nid] + 1)
                queue.append(downstream)

    return [node_map[nid] for nid in sorted_ids], depth
```

**并行分组：** 相同深度（depth）的节点之间无依赖关系，可以并行执行。

```python
def parallel_groups(nodes, edges) -> list[list[NodeDef]]:
    _, depth = topological_sort(nodes, edges)
    groups = {}
    for nid, d in depth.items():
        groups.setdefault(d, []).append(node_map[nid])
    return [groups[d] for d in sorted(groups.keys())]
```

**示例：** 一个 4 节点工作流 `analysis → {dev, test} → doc`
```
depth 0: [analysis]
depth 1: [dev, test]     ← 并行
depth 2: [doc]
```

### 步骤 ③：加载 YAML 模板

`TemplateEngine.load()` 根据节点的 `profile` 加载对应的 YAML 模板文件。

```python
class TemplateEngine:
    def load(self, profile: str) -> dict:
        path = os.path.join(self.template_dir, f"{profile}.yaml")
        if not os.path.isfile(path):
            fallback = os.path.join(self.template_dir, "dev.yaml")
            path = fallback  # 降级回退
        with open(path) as f:
            tmpl = yaml.safe_load(f)
        self._cache[profile] = tmpl  # 缓存
        return tmpl
```

每个 YAML 模板结构：
```yaml
system_prompt: |      # Agent 角色定义
tool_set:             # 可用工具列表（默认 [bash, read, write]）
max_turns: 10         # 最大交互轮数
timeout_s: 120        # 超时秒数
output_format: |      # 输出格式说明（嵌入 prompt）
prompt_template: |    # 提示词模板（含变量占位符）
```

### 步骤 ④：模板变量渲染

`TemplateEngine.render()` 使用正则替换将变量注入 `prompt_template`。

**支持的变量：**

| 变量 | 来源 | 说明 |
|------|------|------|
| `{global.goal}` | `workflow.global_context["goal"]` | 全局目标 |
| `{global.constraints}` | `workflow.global_context["constraints"]` | 约束列表（列表→多行 `- ` 前缀） |
| `{node.label}` | `node.label` | 节点名称 |
| `{params.description}` | `node.desc` | 节点任务描述 |
| `{params.*}` | `node.params.*` | 自定义参数 |
| `{upstream_context}` | 上游节点输出汇总 | 依赖节点的 result 集合 |
| `{sequence}` | 执行序号 | 1, 2, 3... |

**渲染实现：**

    return re.sub(r"\{([a-z_.]+)\}", replace_var, template_str)

### 步骤 ⑤：构建 upstream_context

对于每个节点，从 DAG 边中找出其所有上游（依赖）节点，汇聚它们的输出结果。

```python
def _build_upstream_context(self, node_id, nodes, edges):
    # 找出所有以 node_id 为 target 的 edge
    upstream_ids = [e.source for e in edges if e.target == node_id]
    if not upstream_ids:
        return ""

    parts = []
    for uid in upstream_ids:
        n = node_map.get(uid)
        if n:
            result = n.result or n.desc or ""
            parts.append(f"- **{n.label}**: {result}")
    return "\n".join(parts) if parts else ""
```

**运行时注入：** 执行时，`AgentRunner` 会将上游节点的实际执行结果替换到 `{upstream_context}` 位置（通过 `node.result` 字段）。

## 3. 完整流程伪代码

```python
def compile(self, wf: WorkflowJSON) -> list[PromptTask]:
    # 步骤 ②：拓扑排序
    sorted_nodes, depth = topological_sort(wf.nodes, wf.edges)

    tasks = []
    for i, node in enumerate(sorted_nodes):
        # 步骤 ⑤：构建上游上下文
        upstream_context = self._build_upstream_context(
            node.id, wf.nodes, wf.edges
        )
        # 步骤 ③ + ④ + ⑥
        task = self._compile_node(
            node=node, workflow=wf,
            upstream_context=upstream_context,
            sequence=i + 1, depth=depth.get(node.id, 0),
        )
        tasks.append(task)

    return tasks
```

## 4. 配置参考

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `template_dir` | `./templates/` | YAML 模板目录 |
| `max_turns` (模板) | 10–15 | 节点最大交互轮数 |
| `timeout_s` (模板) | 120–300 | 节点超时秒数 |
| `tool_set` (模板) | `[bash, read, write]` | 可用工具列表 |

## 5. 注意事项

1. **确定性**：Compiler 不调用 LLM，所有操作是纯函数式的
2. **幂等性**：相同的 WorkflowJSON 输入产生相同的 PromptTask[] 输出
3. **模板不存在时**：自动回退到 `dev.yaml`，不会报错终止
4. **变量不存在时**：保留原始占位符 `{key}`，不会渲染为空字符串
5. **上游上下文**：编译时用的是节点初始 `desc`，运行时会被 `AgentRunner` 替换为实际输出
