# 03.1 — WorkflowJSON 数据契约

> **版本: v0.2** — 已与 `agentflow_schema.py` 实际实现对齐。
>
> WorkflowJSON 是 AgentFlow 的工作流定义格式，画布输出此 JSON，经 Compiler 编译为 PromptTask 后执行。

---

## 顶层结构

```json
{
  "version": "1.0.0",
  "workflow_id": "wf-xxx",
  "name": "string",
  "global_context": {
    "goal": "string",
    "language": "zh-CN",
    "constraints": []
  },
  "nodes": [
    {
      "id": "node-1",
      "icon": "🤖",
      "label": "节点名称",
      "desc": "节点描述",
      "color": "blue",
      "profile": "dev",
      "model": null,
      "params": {},
      "result": null,
      "output": null,
      "status": "pending",
      "cost": 0.0,
      "duration": 0,
      "turns": 0,
      "provider": ""
    }
  ],
  "edges": [
    {
      "source": "node-1",
      "target": "node-2"
    }
  ],
  "metadata": {}
}
```

---

## 字段说明

### version

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **默认值** | `"1.0.0"` |
| **⚠️ 对齐说明** | 文档 v0.1 写 `"1.0"`，实际代码默认值为 `"1.0.0"`。✅ **已实现** |

```json
"version": "1.0.0"
```

### workflow_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 小写字母、数字、连字符，通常前缀 `wf-` |
| **⚠️ 对齐说明** | ✅ **已实现** |

```json
"workflow_id": "wf-code-review-pipeline"
```

### name

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | 工作流的人可读名称，用于日志和 UI 展示 |
| **⚠️ 对齐说明** | ✅ **已实现** |

```json
"name": "Code Review Pipeline"
```

### global_context

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{"goal": "", "language": "zh-CN", "constraints": []}` |
| **⚠️ 对齐说明** | 文档 v0.1 为任意 object（如 `project/repo_url`），实际代码使用固定结构 `goal / language / constraints`。✅ **已实现** |

```json
"global_context": {
  "goal": "评审代码变更，生成摘要报告",
  "language": "zh-CN",
  "constraints": ["PEP8", "type hints", "docstrings"]
}
```

### nodes[] — NodeDef

| 项 | 值 |
|------|-----|
| **类型** | `array[NodeDef]` |
| **必填** | 是，至少 1 个 |
| **⚠️ 对齐说明** | 文档 v0.1 使用 `type/name/config/prompt/input_schema/output_schema/timeout_s/retry/on_success/on_failure/tags` 等字段。实际 `NodeDef` 字段已大幅简化并新增执行回填字段。👇 **见下** |

#### 已实现字段（✅ 来自 `agentflow_schema.py`）

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| `id` | `string` | **是** | 节点在 workflow 内唯一标识 | — |
| `icon` | `string` | 否 | 节点图标 emoji | `"🤖"` |
| `label` | `string` | 否 | 节点人可读名称 | `""` |
| `desc` | `string` | 否 | 节点描述 | `""` |
| `color` | `string` | 否 | 节点颜色标识 | `"blue"` |
| `profile` | `string` | 否 | 节点 Profile 类型。枚举：`analysis` / `design` / `dev` / `test` / `doc` / `deploy` | `"dev"` |
| `model` | `string\|null` | 否 | 指定模型名，`null` 表示使用全局默认 | `null` |
| `params` | `object` | 否 | 节点自定义参数（key-value 对，Compiler 注入 prompt） | `{}` |
| `result` | `string\|null` | 否 | **执行回填** — 执行结果摘要（Agent 写入） | `null` |
| `output` | `string\|null` | 否 | **执行回填** — 详细输出（Agent 写入） | `null` |
| `status` | `string` | 否 | **执行回填** — 节点状态。枚举：`pending` / `ready` / `running` / `ok` / `error` / `timeout` / `skipped` | `"pending"` |
| `cost` | `float` | 否 | **执行回填** — 执行成本（美元） | `0.0` |
| `duration` | `int` | 否 | **执行回填** — 执行时长（毫秒） | `0` |
| `turns` | `int` | 否 | **执行回填** — Agent 交互轮数 | `0` |
| `provider` | `string` | 否 | **执行回填** — 服务提供商名 | `""` |

#### 设计意向字段（⏳ 未来版本）

以下字段在文档 v0.1 中出现，但当前代码尚未实现，标记为设计意向：

| 字段 | 说明 | 状态 |
|------|------|------|
| `type` | 节点类型（v0.1 用，当前改为 `profile`） | ⏳ 设计意向 |
| `name` | 节点名称（v0.1 用，当前改为 `label`） | ⏳ 设计意向 |
| `config.model / config.temperature` | 节点级模型配置 | ⏳ 设计意向 |
| `prompt`（模板格式） | 支持 `{{variable}}` 占位符 | ⏳ 设计意向 |
| `input_schema` | 输入数据的 JSON Schema | ⏳ 设计意向 |
| `output_schema` | 输出数据的 JSON Schema | ⏳ 设计意向 |
| `timeout_s` | 节点级超时（当前由 PromptTask 控制） | ⏳ 设计意向 |
| `retry`（对象格式） | 重试策略（当前由 PromptTask 控制） | ⏳ 设计意向 |
| `on_success / on_failure` | 路由钩子（当前由 edges 表达） | ⏳ 设计意向 |
| `tags` | 标签 | ⏳ 设计意向 |

**实际节点示例：**

```json
{
  "id": "node-review-code",
  "icon": "🔍",
  "label": "Review Code Changes",
  "desc": "评审代码变更并生成评审意见",
  "color": "blue",
  "profile": "dev",
  "model": null,
  "params": {
    "diff_content": "实际的 diff 内容",
    "language": "Python"
  },
  "result": null,
  "output": null,
  "status": "pending",
  "cost": 0.0,
  "duration": 0,
  "turns": 0,
  "provider": ""
}
```

**执行后回填示例：**

```json
{
  "id": "node-review-code",
  "icon": "🔍",
  "label": "Review Code Changes",
  "desc": "评审代码变更并生成评审意见",
  "color": "blue",
  "profile": "dev",
  "params": { "diff_content": "...", "language": "Python" },
  "result": "代码质量良好，建议增加类型注解",
  "output": "{\"status\": \"approved\", \"score\": 8.5, ...}",
  "status": "ok",
  "cost": 0.0023,
  "duration": 12345,
  "turns": 3,
  "provider": "openai"
}
```

### edges[] — EdgeDef

| 项 | 值 |
|------|-----|
| **类型** | `array[EdgeDef]` |
| **必填** | 否，默认为空 |
| **⚠️ 对齐说明** | 文档 v0.1 有 `id / condition / data_mapping` 字段。**实际 EdgeDef 只有两个字段：`source` 和 `target`**。`data_mapping` 和 `condition` 标记为设计意向。 |

#### 已实现字段（✅）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `source` | `string` | **是** | 上游节点 ID |
| `target` | `string` | **是** | 下游节点 ID |

```json
{
  "source": "node-review-code",
  "target": "node-summarize"
}
```

#### 设计意向字段（⏳ 未来版本）

| 字段 | 类型 | 说明 | 状态 |
|------|------|------|------|
| `id` | `string` | 边唯一标识 | ⏳ 设计意向 |
| `condition` | `string` | 条件表达式，如 `output.status == 'approved'` | ⏳ 设计意向 |
| `data_mapping` | `object` | 数据映射规则：`{"target_field": "{{source.output.field}}"}` | ⏳ 设计意向 |

---

### metadata

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{}` |
| **⚠️ 对齐说明** | ✅ **已实现**。自由格式，不影响执行逻辑，仅用于管理和展示 |

**常见字段（非强制）：**

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `description` | `string` | 工作流描述 | `"代码评审与自动摘要工作流"` |
| `tags` | `array[string]` | 标签 | `["review", "automation"]` |
| `author` | `string` | 作者 | `"team-alpha"` |
| `created_at` | `string` | 创建时间 (ISO 8601) | `"2026-06-09T10:00:00Z"` |
| `updated_at` | `string` | 更新时间 (ISO 8601) | `"2026-06-09T12:00:00Z"` |
| `version_note` | `string` | 版本说明 | `"Added failure notification node"` |

```json
"metadata": {
  "description": "代码评审与自动摘要工作流",
  "tags": ["review", "automation", "code-quality"],
  "author": "team-alpha",
  "created_at": "2026-06-09T10:00:00Z",
  "updated_at": "2026-06-09T12:00:00Z",
  "version_note": "初始版本"
}
```

## Profile 枚举参考

| 枚举值 | 中文名 | 说明 |
|--------|--------|------|
| `analysis` | 分析型 | 需求分析、问题拆解 |
| `design` | 设计型 | 架构设计、技术方案 |
| `dev` | 开发型 | 编码实现 |
| `test` | 测试型 | 单元测试、集成测试 |
| `doc` | 文档型 | 文档撰写 |
| `deploy` | 部署型 | 部署运维 |

> Profile 在 `agentflow_schema.py` 的 `Profile` 枚举中定义，有 6 个硬编码值。这是当前仅有的 Profile 类型——**不存在节点类型注册表**。

```json

    {

      "desc": "评审代码变更",
      "color": "blue",
      "profile": "dev",
      "params": { "diff_content": "实际的 diff", "language": "Python" },
      "result": null,
      "output": null,
      "status": "pending",
      "cost": 0.0,
      "duration": 0,
      "turns": 0,
      "provider": ""
    },
    {
      "id": "node-summarize",
      "icon": "📝",
      "label": "Generate Review Summary",
      "desc": "生成评审摘要",
      "color": "green",
      "profile": "doc",
      "params": {},
      "result": null,
      "output": null,
      "status": "pending",
      "cost": 0.0,
      "duration": 0,
      "turns": 0,

  ],

    { "source": "node-review-code", "target": "node-summarize" }

    "author": "team-alpha",
    "created_at": "2026-06-09T10:00:00Z"
  }

```
