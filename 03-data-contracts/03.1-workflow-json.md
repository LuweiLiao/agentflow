# 03.1 — WorkflowJSON 数据契约

> WorkflowJSON 是 AgentFlow 的工作流定义格式，面向开发者编写，经 Compiler 编译为 PromptTask 后执行。

---

## 顶层结构

```json
{
  "version": "1.0",
  "workflow_id": "wf-xxx",
  "name": "string",
  "global_context": {},
  "nodes": [],
  "edges": [],
  "metadata": {}
}
```

---

## 字段说明

### version

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 语义化版本号 `MAJOR.MINOR` |
| **说明** | WorkflowJSON schema 版本，用于 Compiler 版本兼容性判断 |

```json
"version": "1.0"
```

### workflow_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 小写字母、数字、连字符，前缀 `wf-` |
| **说明** | 工作流的全局唯一标识；与 edge 引用一致 |

```json
"workflow_id": "wf-code-review-pipeline"
```

### name

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 工作流的人可读名称，用于日志和 UI 展示 |

```json
"name": "Code Review Pipeline"
```

### global_context

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{}` |
| **说明** | 所有节点共享的全局上下文。Compile 时注入到每个节点的 `prompt` 中。可用变量：项目描述、代码库摘要、全局配置等 |

```json
"global_context": {
  "project": "AgentFlow",
  "language": "Python",
  "repo_url": "https://github.com/example/agentflow",
  "review_standards": ["PEP8", "type hints", "docstrings"]
}
```

### nodes[]

| 项 | 值 |
|------|-----|
| **类型** | `array[Node]` |
| **必填** | 是，至少 1 个 |
| **说明** | 工作流中的节点集合。每个节点代表一个执行步骤 |

**节点字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `string` | 是 | 节点在 workflow 内唯一标识 |
| `type` | `string` | 是 | 节点类型，匹配 `templates/{type}.yaml` |
| `name` | `string` | 否 | 节点人可读名称 |
| `config` | `object` | 否 | 节点级配置，覆盖模板默认值 |
| `prompt` | `string` | 是 | 节点提示词模板，支持 `{{variable}}` 占位符 |
| `input_schema` | `object` | 否 | 输入数据的 JSON Schema，支持 `$ref` |
| `output_schema` | `object` | 否 | 输出数据的 JSON Schema，支持 `$ref` |
| `timeout_s` | `integer` | 否 | 节点超时秒数，默认 120 |
| `retry` | `object` | 否 | 重试策略 |
| `on_success` | `string` | 否 | 成功时转发到的节点 ID |
| `on_failure` | `string` | 否 | 失败时转发到的节点 ID |
| `tags` | `array[string]` | 否 | 标签，用于过滤和监控 |

```json
{
  "id": "node-review-code",
  "type": "code-review",
  "name": "Review Code Changes",
  "config": {
    "model": "gpt-4",
    "temperature": 0.2
  },
  "prompt": "请评审以下代码变更：\n```\n{{diff_content}}\n```\n项目语言：{{language}}\n评审标准：{{review_standards}}",
  "input_schema": {
    "type": "object",
    "properties": {
      "diff_content": { "type": "string" },
      "language": { "type": "string" }
    },
    "required": ["diff_content"]
  },
  "output_schema": {
    "$ref": "schemas/code-review-result.json"
  },
  "timeout_s": 300,
  "retry": {
    "max_attempts": 3,
    "backoff": "exponential",
    "initial_delay_s": 5
  },
  "on_success": "node-summarize",
  "on_failure": "node-notify-failure",
  "tags": ["review", "code"]
}
```

#### retry 对象

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| `max_attempts` | `integer` | 否 | 最大重试次数 | `3` |
| `backoff` | `string` | 否 | 退避策略：`fixed` / `exponential` | `exponential` |
| `initial_delay_s` | `integer` | 否 | 首次重试延迟秒数 | `5` |

#### input_schema / output_schema 与 $ref

支持两种写法：

**内联定义：**
```json
"input_schema": {
  "type": "object",
  "properties": {
    "diff_content": { "type": "string" }
  },
  "required": ["diff_content"]
}
```

**$ref 引用：**
```json
"output_schema": {
  "$ref": "schemas/code-review-result.json"
}
```

> `$ref` 路径解析规则：
> - 相对路径：相对于 WorkflowJSON 文件所在目录
> - 绝对路径：文件系统绝对路径
> - URL：远程 schema 地址（需在 Compiler 配置中启用网络访问）
>
> Compiler 在编译 PromptTask 时将 `$ref` 内联展开为完整的 JSON Schema。

---

### edges[]

| 项 | 值 |
|------|-----|
| **类型** | `array[Edge]` |
| **必填** | 否，默认为空 |
| **说明** | 节点间有向边集合。定义执行顺序和数据流 |

**边字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `string` | 否 | 边唯一标识，自动生成时可省略 |
| `source` | `string` | 是 | 源节点 ID |
| `target` | `string` | 是 | 目标节点 ID |
| `condition` | `string` | 否 | 条件表达式，仅当条件成立时执行该边 |
| `data_mapping` | `object` | 否 | 数据映射规则：将 source 输出映射到 target 输入 |

```json
{
  "id": "edge-review-to-summary",
  "source": "node-review-code",
  "target": "node-summarize",
  "condition": "output.status == 'approved'",
  "data_mapping": {
    "review_summary": "{{output.review_summary}}",
    "risk_level": "{{output.risk_level}}"
  }
}
```

#### condition 表达式

条件表达式使用简单 DSL：

| 表达式 | 含义 |
|--------|------|
| `output.status == 'approved'` | 源节点输出状态为 approved |
| `output.score >= 0.8` | 源节点输出分数 >= 0.8 |
| `context.env == 'production'` | 全局上下文中的环境为 production |
| 空字符串 / 未设置 | 无条件，始终执行 |

---

### metadata

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{}` |
| **说明** | 工作流元数据，不影响执行逻辑，仅用于管理和展示 |

**常见字段：**

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `description` | `string` | 工作流描述 | `"代码评审与自动摘要工作流"` |
| `tags` | `array[string]` | 标签 | `["review", "automation"]` |
| `author` | `string` | 作者 | `"team-alpha"` |
| `created_at` | `string` | 创建时间 (ISO 8601) | `"2026-06-09T10:00:00Z"` |
| `updated_at` | `string` | 更新时间 (ISO 8601) | `"2026-06-09T12:00:00Z"` |
| `version_note` | `string` | 版本说明 | `"Added failure notification node"` |

```json
"metadata": {
  "description": "代码评审与自动摘要工作流",
  "tags": ["review", "automation", "code-quality"],
  "author": "team-alpha",
  "created_at": "2026-06-09T10:00:00Z",
  "updated_at": "2026-06-09T12:00:00Z",
  "version_note": "Added failure notification node"
}
```

## 完整示例

```json
{
  "version": "1.0",
  "workflow_id": "wf-code-review-pipeline",
  "name": "Code Review Pipeline",
  "global_context": {
    "project": "AgentFlow",
    "language": "Python",
    "review_standards": ["PEP8", "type hints", "docstrings", "security"]
  },
  "nodes": [
    {
      "id": "node-review-code",
      "type": "code-review",
      "name": "Review Code Changes",
      "prompt": "请严格按以下标准评审代码变更：\n```\n{{diff_content}}\n```\n语言：{{language}}\n标准：{{review_standards}}",
      "input_schema": {
        "type": "object",
        "properties": {
          "diff_content": { "type": "string" }
        },
        "required": ["diff_content"]
      },
      "output_schema": {
        "$ref": "schemas/code-review-result.json"
      },
      "timeout_s": 300,
      "retry": {
        "max_attempts": 3,
        "backoff": "exponential",
        "initial_delay_s": 5
      }
    },
    {
      "id": "node-summarize",
      "type": "summarizer",
      "name": "Generate Review Summary",
      "prompt": "根据以下评审结果生成简洁摘要：\n{{review_summary}}",
      "timeout_s": 60
    },
    {
      "id": "node-notify-failure",
      "type": "notifier",
      "name": "Notify Review Failure",
      "prompt": "评审失败，请通知相关人员：\n错误信息：{{error}}",
      "timeout_s": 30
    }
  ],
  "edges": [
    {
      "id": "edge-review-to-summary",
      "source": "node-review-code",
      "target": "node-summarize",
      "condition": "output.status == 'approved'",
      "data_mapping": {
        "review_summary": "{{output.review_summary}}"
      }
    },
    {
      "id": "edge-review-to-notify",
      "source": "node-review-code",
      "target": "node-notify-failure",
      "condition": "output.status == 'failed'",
      "data_mapping": {
        "error": "{{output.error_message}}"
      }
    }
  ],
  "metadata": {
    "description": "代码评审与自动摘要工作流",
    "tags": ["review", "automation", "code-quality"],
    "author": "team-alpha",
    "created_at": "2026-06-09T10:00:00Z"
  }
}
```
