# 03.2 — PromptTask 数据契约

> PromptTask 是 AgentFlow Compiler 编译 WorkflowJSON 后产出的可执行单元。每个 PromptTask 包含完整的提示词、执行参数和依赖关系，可直接交给执行引擎运行。

---

## 顶层结构

```json
{
  "task_id": "pt-xxx",
  "workflow_id": "wf-xxx",
  "node_id": "node-xxx",
  "node_type": "code-review",
  "sequence": 1,
  "parallel_group": null,
  "depends_on": [],
  "prompt": "string (完整的、已编译的提示词)",
  "output_schema": {},
  "timeout_s": 120,
  "retry": {}
}
```

---

## 字段说明

### task_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 前缀 `pt-` + UUIDv4 或 雪花 ID |
| **说明** | PromptTask 全局唯一标识。每次编译生成新的 task_id |

```json
"task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
```

### workflow_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 来源 WorkflowJSON 的 workflow_id，用于追溯 |

```json
"workflow_id": "wf-code-review-pipeline"
```

### node_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 对应 WorkflowJSON 中的节点 ID |

```json
"node_id": "node-review-code"
```

### node_type

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 节点类型，匹配 `templates/{type}.yaml`。执行引擎据此选择处理器和模型 |

```json
"node_type": "code-review"
```

### sequence

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 是 |
| **说明** | Compiler 根据拓扑排序分配的全局执行序号。序号越小越先执行 |

```json
"sequence": 1
```

**排序规则：**
1. Compiler 使用 nodes[] + edges[] 构建 DAG
2. 进行拓扑排序
3. 按拓扑序分配 `sequence` 序号（从 1 开始递增）

### parallel_group

| 项 | 值 |
|------|-----|
| **类型** | `string \| null` |
| **必填** | 是 |
| **说明** | 并行组标识。属于同一 parallel_group 的节点可并行执行 |
| **默认值** | `null`（表示顺序执行，无并行组） |

```json
"parallel_group": null
```

```json
"parallel_group": "pg-summarize"
```

**分组规则：**
- Compiler 分析 DAG 中无依赖关系的节点，将同一层级的节点归入同一 parallel_group
- 属于同一组的节点互无依赖，执行引擎可并发调度

### depends_on[]

| 项 | 值 |
|------|-----|
| **类型** | `array[string]` |
| **必填** | 是 |
| **说明** | 当前节点依赖的 task_id 列表。所有依赖完成后该节点才会被调度 |
| **默认值** | `[]`（无依赖，可直接执行） |

```json
"depends_on": ["pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"]
```

**依赖推导规则：**
- Compiler 遍历 edges[]，对于每条 `edge(source → target)`，找到 source 对应的 PromptTask 的 task_id
- 将该 task_id 添加到 target 对应 PromptTask 的 `depends_on[]`

**条件边处理：**
- 如果 edge 有 `condition`，则依赖关系仍然建立，但执行引擎会检查条件再处理结果
- 条件不满足时视为"跳过"，当前节点仍正常调度

### prompt

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | **完整的、已编译的提示词。** 这是 WorkflowJSON 中 `global_context` + `node.prompt` 拼接和内联变量替换后的结果 |

```json
"prompt": "请严格按以下标准评审代码变更：\n```\n@@ -1,3 +1,5 @@\n-def add(a, b):\n-    return a + b\n+def add(a, b):\n+    \"\"\"Add two numbers.\"\"\"\n+    return a + b\n```\n语言：Python\n标准：PEP8, type hints, docstrings, security"
```

**编译过程（Compiler 做了这些）：**

```
原始 WorkflowJSON node.prompt:
  "请评审以下代码变更：\n```\n{{diff_content}}\n```\n语言：{{language}}"

         │
         ▼
Step 1: 注入 global_context 变量
  global_context.language → "Python"
  → "请评审以下代码变更：\n```\n{{diff_content}}\n```\n语言：Python"

         │
         ▼
Step 2: 替换输入数据占位符
  输入数据中的 diff_content → "实际的 diff 内容"
  → "请评审以下代码变更：\n```\n@@ -1,3 +1,5 @@\n...\n```\n语言：Python"

         │
         ▼
Step 3: 字符串模板编译
  替换 {{variable}} 格式的所有占位符
  最终结果写入 prompt 字段
```

### output_schema

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否 |
| **说明** | 输出数据的 JSON Schema。已经过 Compiler 内联展开（`$ref` 已解析） |

```json
"output_schema": {
  "type": "object",
  "properties": {
    "status": {
      "type": "string",
      "enum": ["approved", "changes_requested", "failed"]
    },
    "review_summary": { "type": "string" },
    "issues": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "severity": { "type": "string", "enum": ["critical", "major", "minor", "info"] },
          "description": { "type": "string" },
          "line_number": { "type": "integer" }
        }
      }
    },
    "score": { "type": "number", "minimum": 0, "maximum": 10 },
    "error_message": { "type": "string" }
  },
  "required": ["status", "review_summary"]
}
```

### timeout_s

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 是 |
| **说明** | 节点执行超时秒数。从 WorkflowJSON node.timeout_s 继承，未设置时使用默认值 |
| **默认值** | `120` |

```json
"timeout_s": 300
```

### retry

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 是 |
| **说明** | 重试策略。从 WorkflowJSON node.retry 继承，未设置时使用默认值 |

```json
"retry": {
  "max_attempts": 3,
  "backoff": "exponential",
  "initial_delay_s": 5
}
```

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| `max_attempts` | `integer` | 是 | 最大重试次数 | `3` |
| `backoff` | `string` | 是 | 退避策略：`fixed` / `exponential` | `exponential` |
| `initial_delay_s` | `integer` | 是 | 首次重试延迟秒数 | `5` |

## Compiler 编译流程全景

```
WorkflowJSON                          PromptTask[]
    │                                      │
    ▼                                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    Compiler                                  │
│                                                              │
│  Step 1 ── 解析 WorkflowJSON                                 │
│    ├── 校验 version 兼容性                                   │
│    ├── 检查 nodes[] 完整性                                   │
│    └── 检查 edges[] 连通性                                   │
│                                                              │
│  Step 2 ── 构建 DAG                                          │
│    ├── 以 nodes[] 为顶点                                     │
│    ├── 以 edges[] 为有向边                                   │
│    ├── 检测环（有环 → 编译失败）                             │
│    └── 检测孤立节点（无入度 + 出度 → 警告）                  │
│                                                              │
│  Step 3 ── 拓扑排序 & 分组                                   │
│    ├── 拓扑排序 → 分配 sequence 序号                         │
│    ├── 同一层无依赖节点 → 同一 parallel_group                │
│    └── 每个节点收集 depends_on[]                             │
│                                                              │
│  Step 4 ── 编译 prompt                                       │
│    ├── 读取 global_context                                   │
│    ├── 拼接全局变量到节点 prompt                             │
│    ├── 替换 {{variable}} 占位符                              │
│    └── 生成完整 prompt 字符串                                │
│                                                              │
│  Step 5 ── 解析 Schema                                       │
│    ├── 解析 input_schema / output_schema 中的 $ref           │
│    ├── 内联展开为完整 JSON Schema                            │
│    └── 注入 output_schema 到 PromptTask                      │
│                                                              │
│  Step 6 ── 继承执行参数                                      │
│    ├── timeout_s (节点定义 → 默认)                           │
│    ├── retry (节点定义 → 默认)                               │
│    ├── on_success → 转化为 edges                             │
│    └── on_failure → 转化为 edges                             │
│                                                              │
│  Step 7 ── 生成 PromptTask 数组                              │
│    ├── 为每个节点生成一个 PromptTask 对象                    │
│    ├── 分配唯一 task_id (UUID)                               │
│    └── 输出 PromptTask[]                                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
