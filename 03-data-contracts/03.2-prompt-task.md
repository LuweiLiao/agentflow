# 03.2 — PromptTask 数据契约

> **版本: v0.2** — 已与 `agentflow_schema.py` 实际实现对齐。
>
> PromptTask 是 AgentFlow Compiler 编译 WorkflowJSON 后产出的可执行单元。每个 PromptTask 包含完整的提示词、System Prompt、工具集和执行参数，可直接交给 Orchestrator 运行。

---

## 顶层结构

```json
{
  "task_id": "pt-xxx",
  "workflow_id": "wf-xxx",
  "node_id": "node-xxx",
  "node_type": "dev",
  "sequence": 1,
  "parallel_group": 0,
  "depends_on": [],
  "prompt": "string (完整的、已编译的提示词)",
  "system_prompt": "string (Agent 系统提示词)",
  "tool_set": ["bash", "read", "write"],
  "max_turns": 10,
  "timeout_s": 120,
  "retry": 0
}
```

---

## 字段说明

### task_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | **是** |
| **格式** | 通常前缀 `pt-` + 标识符 |
| **⚠️ 对齐说明** | ✅ **已实现**。每次编译生成新的 task_id |

```json
"task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
```

### workflow_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | 来源 WorkflowJSON 的 workflow_id，用于追溯 |
| **⚠️ 对齐说明** | ✅ **已实现** |

```json
"workflow_id": "wf-code-review-pipeline"
```

### node_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | 对应 WorkflowJSON 中的节点 ID |
| **⚠️ 对齐说明** | ✅ **已实现** |

```json
"node_id": "node-review-code"
```

### node_type

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `"dev"` |
| **说明** | 节点 Profile 类型，取值同 `Profile` 枚举：`analysis` / `design` / `dev` / `test` / `doc` / `deploy`。执行引擎据此选择 Agent 的 System Prompt 和工具集 |
| **⚠️ 对齐说明** | 文档 v0.1 写 `code-review` 等自由字符串。实际使用 6 个硬编码 Profile 值。✅ **已实现** |

```json
"node_type": "dev"
```

### sequence

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否，默认 `0` |
| **说明** | Compiler 根据拓扑排序分配的全局执行序号。序号越小越先执行 |
| **⚠️ 对齐说明** | ✅ **已实现** |

```json
"sequence": 1
```

**排序规则：**
1. Compiler 使用 nodes[] + edges[] 构建 DAG
2. 进行拓扑排序（Kahn 算法）
3. 按拓扑序分配 `sequence` 序号（从 0 开始递增）

### parallel_group

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否，默认 `0` |
| **说明** | 并行组 ID。同组节点无依赖关系，可并行执行。`0` 表示不分组 |
| **⚠️ 对齐说明** | 文档 v0.1 为 `string\|null`（如 `"pg-summarize"`）。实际代码为 `int` 类型，默认 `0`。✅ **已实现** |

```json
"parallel_group": 0
```

```json
"parallel_group": 1
```

**分组规则：**
- Compiler 利用 `build_dag` 和 `topological_sort` 函数计算 DAG 的深度层级
- 同层节点（无依赖关系）被分配到同一 `parallel_group`
- 执行引擎对同组节点可并发调度

### depends_on[]

| 项 | 值 |
|------|-----|
| **类型** | `array[string]` |
| **必填** | 否，默认 `[]` |
| **说明** | 当前任务依赖的 node_id 列表（⚠️ 注意：依赖的是 node_id，不是 task_id）。所有依赖完成后该任务才会被调度 |
| **⚠️ 对齐说明** | ✅ **已实现**。文档 v0.1 写依赖 task_id，实际代码中 `depends_on` 存的是 `node_id[]`，Orchestrator 按 node_id 匹配 |

```json
"depends_on": ["node-review-code"]
```

**依赖推导规则：**
- Compiler 遍历 edges[]，对于每条 `edge(source → target)`，将 source 的 node_id 加入 target 对应的 `depends_on[]`

### prompt

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | **完整的、已编译的提示词**。Compiler 将 WorkflowJSON 中 `global_context` 和 `node.params` 拼接渲染后的结果 |
| **⚠️ 对齐说明** | 文档 v0.1 描述了复杂的 3 步编译过程（注入 global_context → 替换输入占位符 → 字符串模板）。实际为扁平 `str` 字段，不包含 `output_schema`。✅ **已实现** |

```json
"prompt": "请严格按以下标准评审代码变更：\n```\n@@ -1,3 +1,5 @@\n-def add(a, b):\n-    return a + b\n+def add(a, b):\n+    \"\"\"Add two numbers.\"\"\"\n+    return a + b\n```\n语言：Python\n标准：PEP8, type hints, docstrings, security"
```

### system_prompt

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | Agent 系统提示词。根据 `node_type`（Profile）从模板选择，控制 Agent 角色和行为 |
| **⚠️ 对齐说明** | 文档 v0.1 未提及此字段。🔍 **新增已实现字段** |

```json
"system_prompt": "你是一个资深的代码评审专家。请严格按照 PEP8 标准评审代码..."
```

### tool_set

| 项 | 值 |
|------|-----|
| **类型** | `array[string]` |
| **必填** | 否，默认 `["bash", "read", "write"]` |
| **说明** | Agent 可用的工具列表。工具名对应已注册的工具实现 |
| **⚠️ 对齐说明** | 文档 v0.1 未提及此字段。🔍 **新增已实现字段** |

| 工具名 | 说明 |
|--------|------|
| `bash` | 执行 Shell 命令 |
| `read` | 读取文件内容 |
| `write` | 写入文件内容 |

```json
"tool_set": ["bash", "read", "write"]
```

### max_turns

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否，默认 `10` |
| **说明** | Agent 最大交互轮数。达到后强制结束 |
| **⚠️ 对齐说明** | 文档 v0.1 未提及此字段。🔍 **新增已实现字段** |

```json
"max_turns": 10
```

### timeout_s

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否，默认 `120` |
| **说明** | 节点执行超时秒数 |
| **⚠️ 对齐说明** | ✅ **已实现**。默认值 120 |

```json
"timeout_s": 300
```

### retry

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否，默认 `0` |
| **说明** | 重试次数。`0` 表示不重试 |
| **⚠️ 对齐说明** | 文档 v0.1 为完整对象 `{max_attempts, backoff, initial_delay_s}`。**实际为 `int` 类型**，仅表示重试次数。复杂的重试策略（指数退避等）标记为设计意向 |

```json
"retry": 0
```

```json
"retry": 3
```

---

## 设计意向字段（⏳ 未来版本）

以下字段在文档 v0.1 中出现但当前代码尚未实现：

| 字段 | 说明 | 文档 v0.1 | 实际状态 |
|------|------|-----------|----------|
| `output_schema` | 输出数据的 JSON Schema | 有完整 examples | ⏳ 设计意向 |
| `retry`（对象） | `max_attempts / backoff / initial_delay_s` | 详细定义 | ⏳ 已简化为 `int` |
| `parallel_group`（string） | 字符串形式的分组标识 | `"pg-summarize"` | ⏳ 当前为 `int` 类型 |

## Compiler 编译流程全景

```
WorkflowJSON                          PromptTask[]
    │                                      │
    ▼                                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    Compiler                                  │
│                                                              │
│  Step 1 ── 解析 WorkflowJSON                                 │
│    ├── 校验 version 兼容性                                   │
│    ├── 检查 nodes[] 完整性                                   │
│    └── 检查 edges[] 连通性                                   │
│                                                              │
│  Step 2 ── 构建 DAG                                          │
│    ├── 以 nodes[] 为顶点                                     │
│    ├── 以 edges[] 为有向边                                   │
│    ├── 检测环（有环 → 编译失败）                             │
│    └── 检测孤立节点                                          │
│                                                              │
│  Step 3 ── 拓扑排序 & 分组                                   │
│    ├── 拓扑排序 → 分配 sequence 序号                         │
│    ├── 同一层无依赖节点 → 同一 parallel_group                │
│    └── 每个节点收集 depends_on[]                             │
│                                                              │
│  Step 4 ── 编译 prompt                                       │
│    ├── 读取 global_context                                   │
│    ├── 拼接全局变量 + node.params → 完整 prompt             │
│    └── 生成完整 prompt 字符串                                │
│                                                              │
│  Step 5 ── 设置执行参数                                      │
│    ├── 根据 node_type 选择 system_prompt                     │
│    ├── 注入 tool_set (依赖 Profile)                          │
│    ├── 设置 max_turns / timeout_s / retry                    │
│    └── 生成 PromptTask 对象                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
