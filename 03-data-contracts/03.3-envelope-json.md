# 03.3 — Envelope JSON 数据契约

> **版本: v0.2** — 已与 `agentflow_schema.py` 实际实现对齐。
>
> Envelope JSON 是 AgentFlow 标准化的执行结果封装格式，用于 Agent 输出 → Orchestrator 汇聚 → 画布回填。

---

## 顶层结构

```json
{
  "task_id": "pt-xxx",
  "node_id": "node-xxx",
  "status": "ok",
  "summary": "执行摘要",
  "payload": {},
  "metrics": {
    "cost": 0.0,
    "duration_ms": 0,
    "prompt_tokens": 0,
    "completion_tokens": 0,
    "turns": 0,
    "model": "",
    "provider": "",
    "files": []
  },
  "errors": [],
  "timestamp": 0.0
}
```

---

## 字段说明

### task_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | **是** |
| **说明** | 来源 PromptTask 的 task_id，链接到 PromptTask 契约 |
| **⚠️ 对齐说明** | 文档 v0.1 无此顶层字段（在 `source.task_id` 中）。实际为顶层必填字段。✅ **已实现** |

```json
"task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
```

### node_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | 对应 WorkflowJSON 中的节点 ID |
| **⚠️ 对齐说明** | 文档 v0.1 无此字段。🔍 **新增已实现字段** |

```json
"node_id": "node-review-code"
```

### status

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `"pending"` |
| **说明** | 节点执行状态枚举 |
| **⚠️ 对齐说明** | 文档 v0.1 有 `ok / error / timeout / skipped / partial / cancelled`。实际枚举值同 `NodeStatus`：`pending` / `ready` / `running` / `ok` / `error` / `timeout` / `skipped`。✅ **已实现** |

**完整枚举值（来自 `NodeStatus`）：**

| 枚举值 | 说明 |
|--------|------|
| `pending` | 等待执行 |
| `ready` | 就绪可执行 |
| `running` | 正在执行 |
| `ok` | 执行成功 |
| `error` | 执行出错 |
| `timeout` | 执行超时 |
| `skipped` | 任务被跳过 |

```json
"status": "ok"
```

### summary

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否，默认 `""` |
| **说明** | 执行结果摘要。Orchestrator 将其回填到 `NodeDef.result` |
| **⚠️ 对齐说明** | 文档 v0.1 无此字段（仅 `payload.data`）。🔍 **新增已实现字段** |

```json
"summary": "代码质量良好，符合 PEP8 标准。建议在公共函数上增加类型注解。"
```

### payload

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{}` |
| **说明** | Agent 原始输出。自由格式 dict，Orchestrator 可将其中 `output` 字段回填到 `NodeDef.output` |
| **⚠️ 对齐说明** | 文档 v0.1 有 `content_type / schema_ref / data` 子结构。实际为简单 `dict`，无强制子字段。✅ **已实现** |

```json
"payload": {
  "output": "{\"status\": \"approved\", \"score\": 8.5, \"issues\": [...]}"
}
```

### metrics — ResultMetrics

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `ResultMetrics()` |
| **⚠️ 对齐说明** | 文档 v0.1 有 `tokens_used {prompt/completion/total}` 和 `elapsed_ms`。实际为 `ResultMetrics` 对象，字段不同。👇 **见下** |

#### 已实现字段（✅）

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `cost` | `float` | 执行成本（美元） | `0.0` |
| `duration_ms` | `int` | 执行耗时（毫秒） | `0` |
| `prompt_tokens` | `int` | 提示词 Token 数 | `0` |
| `completion_tokens` | `int` | 输出 Token 数 | `0` |
| `turns` | `int` | Agent 交互轮数 | `0` |
| `model` | `string` | 使用的模型标识 | `""` |
| `provider` | `string` | 服务提供商 | `""` |
| `files` | `array[string]` | 生成的文件路径列表 | `[]` |

```json
"metrics": {
  "cost": 0.0023,
  "duration_ms": 12345,
  "prompt_tokens": 1532,
  "completion_tokens": 489,
  "turns": 3,
  "model": "gpt-4-0613",
  "provider": "openai",
  "files": ["/tmp/report.md"]
}
```

### errors[]

| 项 | 值 |
|------|-----|
| **类型** | `array[string]` |
| **必填** | 否，默认 `[]` |
| **说明** | 错误消息列表。`status` 为 `error` / `timeout` 时填充 |
| **⚠️ 对齐说明** | 文档 v0.1 无此字段（错误信息在 `payload.data` 中）。🔍 **新增已实现字段** |

```json
"errors": ["LLM returned invalid JSON: unterminated string at line 5"]
```

### timestamp

| 项 | 值 |
|------|-----|
| **类型** | `float` |
| **必填** | 否，默认 `0.0` |
| **说明** | Unix 时间戳（秒），消息生成时间 |
| **⚠️ 对齐说明** | 文档 v0.1 为 ISO 8601 字符串，位于 `envelope.timestamp`。实际为 `float` 类型 Unix 时间戳。✅ **已实现** |

```json
"timestamp": 1750366245.123
```

## 设计意向字段（⏳ 未来版本）

以下结构在文档 v0.1 中出现但当前代码尚未实现：

| 字段路径 | 说明 | 文档 v0.1 | 实际状态 |
|----------|------|-----------|----------|
| `envelope.message_id` | 消息全局唯一标识 | 前缀 `msg-` + UUID | ⏳ 设计意向 |
| `envelope.correlation_id` | 关联 ID | 前缀 `corr-` + UUID | ⏳ 设计意向 |
| `envelope.type` | 消息类型枚举 | 14 种枚举值 | ⏳ 设计意向 |
| `source.agent_id` | Agent 标识 | 前缀 `agent-` | ⏳ 设计意向 |
| `source.task_id` | 来源 task_id | 字符串 | ⏳ 已在顶层实现 |
| `payload.content_type` | 负载内容类型 | MIME / 自定义 | ⏳ 设计意向 |
| `payload.schema_ref` | Schema 引用路径 | 字符串 | ⏳ 设计意向 |
| `payload.data` | 实际数据内容 | 任意 JSON | ⏳ 设计意向 |

> **设计说明**：当前 EnvelopeJSON 采用扁平结构（无 `envelope` / `source` 包装），以简化 Agent 的实现负担。`payload` 作为自由格式 dict，Agent 可直接写入任意结构化输出。未来的消息队列集成可能引入完整的信封包装。

```json

  "task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC",
  "node_id": "node-review-code",

    "provider": "openai",

```

## to_node_dict() 回填映射

`EnvelopeJSON` 提供 `to_node_dict()` 方法，将执行结果转换回 `NodeDef` 兼容格式供前端消费：

| EnvelopeJSON 源 | → | NodeDef 目标 |
|-----------------|---|-------------|
| `node_id` | → | `id` |
| `summary` | → | `result` |
| `payload.get("output", "")` | → | `output` |
| `status` | → | `status` |
| `metrics.cost` | → | `cost` |
| `metrics.duration_ms` | → | `duration` |
| `metrics.turns` | → | `turns` |
| `metrics.model` | → | `model` |
| `metrics.provider` | → | `provider` |
| `metrics.files` | → | `files` |
