# 03.3 — Envelope JSON 数据契约

> Envelope JSON 是 AgentFlow 标准化的执行结果封装格式，用于 Agent 之间、Agent 到 Logging 系统之间的结果传输和持久化。

---

## 顶层结构

```json
{
  "envelope": {},
  "source": {},
  "payload": {},
  "status": "ok",
  "metrics": {}
}
```

---

## 字段说明

### envelope

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 是 |
| **说明** | 消息信封，包含消息路由和追踪所需的所有元数据 |

#### envelope.message_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 前缀 `msg-` + UUIDv4 |
| **说明** | 消息全局唯一标识 |

```json
"message_id": "msg-01J2XYK0R0K8X7VQZ2W5N6P9QE"
```

#### envelope.correlation_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 前缀 `corr-` + UUIDv4 |
| **说明** | 关联 ID，用于追踪一组相关消息。同一工作流执行的所有消息共享同一个 correlation_id |

```json
"correlation_id": "corr-01J2XYK0R0K8X7VQZ2W5N6P9QF"
```

#### envelope.type

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 消息类型枚举，标识消息的用途和语义 |

**完整枚举值：**

| 枚举值 | 说明 | 方向 |
|--------|------|------|
| `task.submitted` | 任务已提交，等待执行 | Agent → Engine |
| `task.started` | 任务开始执行 | Agent → Engine |
| `task.completed` | 任务执行完成（成功） | Agent → Engine |
| `task.failed` | 任务执行失败 | Agent → Engine |
| `task.timeout` | 任务执行超时 | Agent → Engine |
| `task.skipped` | 任务被跳过（条件边不满足） | Agent → Engine |
| `task.retrying` | 任务正在重试 | Agent → Engine |
| `workflow.completed` | 整个工作流执行完成 | Engine → Agent |
| `workflow.failed` | 整个工作流执行失败 | Engine → Agent |
| `log.info` | 信息日志 | Agent → Logging |
| `log.warning` | 警告日志 | Agent → Logging |
| `log.error` | 错误日志 | Agent → Logging |
| `heartbeat` | 心跳消息（用于检测 Agent 活性） | Agent → Monitor |
| `system.error` | 系统内部错误 | Any → Monitor |

```json
"type": "task.completed"
```

#### envelope.timestamp

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | ISO 8601，UTC 时间，精确到毫秒 |
| **说明** | 消息生成时间戳 |

```json
"timestamp": "2026-06-09T10:30:45.123Z"
```

**完整 envelope 示例：**

```json
"envelope": {
  "message_id": "msg-01J2XYK0R0K8X7VQZ2W5N6P9QE",
  "correlation_id": "corr-01J2XYK0R0K8X7VQZ2W5N6P9QF",
  "type": "task.completed",
  "timestamp": "2026-06-09T10:30:45.123Z"
}
```

---

### source

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 是 |
| **说明** | 消息来源信息，标识消息的发出者和所属任务 |

#### source.agent_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **格式** | 前缀 `agent-` + 标识符 |
| **说明** | 发出消息的 Agent 标识 |

```json
"agent_id": "agent-reviewer-01"
```

#### source.task_id

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 来源 PromptTask 的 task_id，链接到 PromptTask 契约 |

```json
"task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
```

**完整 source 示例：**

```json
"source": {
  "agent_id": "agent-reviewer-01",
  "task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
}
```

---

### payload

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 是 |
| **说明** | 消息负载，包含实际的数据内容 |

#### payload.content_type

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 负载内容类型 MIME 或 自定义类型标识 |

```json
"content_type": "application/json"
```

**常见值：**

| content_type | 说明 |
|-------------|------|
| `application/json` | JSON 格式数据 |
| `text/plain` | 纯文本数据 |
| `application/x-yaml` | YAML 格式数据 |
| `x-agentflow/code-review-result` | 自定义 AgentFlow 类型 |
| `application/octet-stream` | 二进制数据 |

#### payload.schema_ref

| 项 | 值 |
|------|-----|
| **类型** | `string \| null` |
| **必填** | 否 |
| **说明** | 数据 schema 引用路径。如果 payload.data 遵循某个 JSON Schema，则在此声明引用 |
| **默认值** | `null` |

```json
"schema_ref": "schemas/code-review-result.json"
```

#### payload.data

| 项 | 值 |
|------|-----|
| **类型** | `any` |
| **必填** | 是 |
| **说明** | 实际的数据内容。类型由 content_type 决定。通常为 JSON object |

```json
"data": {
  "status": "approved",
  "review_summary": "代码质量良好，符合 PEP8 标准。建议在公共函数上增加类型注解。",
  "issues": [
    {
      "severity": "minor",
      "description": "函数 add 缺少类型注解",
      "line_number": 1
    },
    {
      "severity": "info",
      "description": "建议增加错误处理",
      "line_number": 3
    }
  ],
  "score": 8.5
}
```

**完整 payload 示例：**

```json
"payload": {
  "content_type": "application/json",
  "schema_ref": "schemas/code-review-result.json",
  "data": {
    "status": "approved",
    "review_summary": "代码质量良好，符合 PEP8 标准。",
    "issues": [
      { "severity": "minor", "description": "缺少类型注解", "line_number": 1 }
    ],
    "score": 8.5
  }
}
```

---

### status

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 是 |
| **说明** | 任务执行状态枚举 |

**完整枚举值：**

| 枚举值 | 说明 |
|--------|------|
| `ok` | 执行成功，payload.data 包含有效结果 |
| `error` | 执行出错，payload.data 包含错误信息 |
| `timeout` | 执行超时 |
| `skipped` | 任务被跳过（条件边不满足） |
| `partial` | 部分成功（多模型输出时，部分模型成功部分失败） |
| `cancelled` | 任务被取消 |

```json
"status": "ok"
```

---

### metrics

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否，默认 `{}` |
| **说明** | 执行指标，用于性能监控和成本计算 |

#### metrics.tokens_used

| 项 | 值 |
|------|-----|
| **类型** | `object` |
| **必填** | 否 |
| **说明** | Token 使用统计 |

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `prompt` | `integer` | 提示词 Token 数 |
| `completion` | `integer` | 输出 Token 数 |
| `total` | `integer` | 总计 Token 数 |

```json
"tokens_used": {
  "prompt": 1532,
  "completion": 489,
  "total": 2021
}
```

#### metrics.elapsed_ms

| 项 | 值 |
|------|-----|
| **类型** | `integer` |
| **必填** | 否 |
| **说明** | 执行耗时，单位毫秒 |

```json
"elapsed_ms": 12345
```

#### metrics.model

| 项 | 值 |
|------|-----|
| **类型** | `string` |
| **必填** | 否 |
| **说明** | 使用的模型标识 |

```json
"model": "gpt-4-0613"
```

**完整 metrics 示例：**

```json
"metrics": {
  "tokens_used": {
    "prompt": 1532,
    "completion": 489,
    "total": 2021
  },
  "elapsed_ms": 12345,
  "model": "gpt-4-0613"
}
```

## 完整示例

### 成功执行

```json
{
  "envelope": {
    "message_id": "msg-01J2XYK0R0K8X7VQZ2W5N6P9QE",
    "correlation_id": "corr-01J2XYK0R0K8X7VQZ2W5N6P9QF",
    "type": "task.completed",
    "timestamp": "2026-06-09T10:30:45.123Z"
  },
  "source": {
    "agent_id": "agent-reviewer-01",
    "task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
  },
  "payload": {
    "content_type": "application/json",
    "schema_ref": "schemas/code-review-result.json",
    "data": {
      "status": "approved",
      "review_summary": "代码质量良好，符合 PEP8 标准。",
      "issues": [
        {
          "severity": "minor",
          "description": "函数 add 缺少类型注解",
          "line_number": 1
        }
      ],
      "score": 8.5
    }
  },
  "status": "ok",
  "metrics": {
    "tokens_used": {
      "prompt": 1532,
      "completion": 489,
      "total": 2021
    },
    "elapsed_ms": 12345,
    "model": "gpt-4-0613"
  }
}
```

### 执行失败

```json
{
  "envelope": {
    "message_id": "msg-01J2XYK0R0K8X7VQZ2W5N6P9QG",
    "correlation_id": "corr-01J2XYK0R0K8X7VQZ2W5N6P9QF",
    "type": "task.failed",
    "timestamp": "2026-06-09T10:31:00.456Z"
  },
  "source": {
    "agent_id": "agent-reviewer-01",
    "task_id": "pt-01J2XYK0R0K8X7VQZ2W5N6P9QC"
  },
  "payload": {
    "content_type": "application/json",
    "data": {
      "error_code": "LLM_RESPONSE_PARSE_ERROR",
      "error_message": "LLM returned invalid JSON: unterminated string at line 5",
      "raw_response": "{\n  \"status\": \"approved\",..."
    }
  },
  "status": "error",
  "metrics": {
    "tokens_used": {
      "prompt": 1532,
      "completion": 42,
      "total": 1574
    },
    "elapsed_ms": 8765,
    "model": "gpt-4-0613"
  }
}
```

```json
    "message_id": "msg-01J2XYK0R0K8X7VQZ2W5N6P9QH",

    "elapsed_ms": 300000,
    "model": "gpt-4-0613"
  }
}
```

## 消息类型枚举完整参考

| 枚举值 | 说明 | 包含 payload.data | 典型 status |
|--------|------|-------------------|-------------|
| `task.submitted` | 任务已提交 | 任务元数据 | — |
| `task.started` | 任务开始执行 | 开始时间、模型信息 | — |
| `task.completed` | 任务成功完成 | 执行结果 | `ok` |
| `task.failed` | 任务执行失败 | 错误信息 | `error` |
| `task.timeout` | 任务超时 | 超时详情 + 部分输出 | `timeout` |
| `task.skipped` | 任务被跳过 | 跳过原因 | `skipped` |
| `task.retrying` | 任务正在重试 | 重试次数、等待时间 | — |
| `workflow.completed` | 工作流完成 | 汇总结果 | `ok` / `partial` |
| `workflow.failed` | 工作流失败 | 失败原因汇总 | `error` |
| `log.info` | 信息日志 | 日志消息 | — |
| `log.warning` | 警告日志 | 日志消息 | — |
| `log.error` | 错误日志 | 日志消息 + 堆栈 | — |
| `heartbeat` | 心跳 | Agent 状态信息 | — |
| `system.error` | 系统内部错误 | 系统错误详情 | — |

## 三契约联动示例

```
WorkflowJSON                  PromptTask[]                    Envelope JSON
─────────────────      ────────────────────      ──────────────────────────────
wf-code-review        pt-01...9QC              msg-01...QE
                     ┌─────────────────┐      ┌──────────────────────────┐
  node-review-code──▶│ task_id         │─────▶│ envelope.type:           │
  (code-review)      │ node_id         │      │   task.completed         │
                     │ prompt: ...     │      │ source.task_id:          │
                     │ depends_on: []  │      │   pt-01...9QC            │
                     │ sequence: 1     │      │ payload.data: {...}      │
                     └─────────────────┘      │ status: ok               │
                                              │ metrics.tokens_used:...  │
  node-summarize     pt-01...9QD              └──────────────────────────┘
  (summarizer)      ┌─────────────────┐      ┌──────────────────────────┐
                     │ task_id         │─────▶│ envelope.type:           │
                     │ node_id         │      │   task.completed         │
                     │ prompt: ...     │      │ source.task_id:          │
                     │ depends_on:     │      │   pt-01...9QD            │
                     │   [pt-01...9QC] │      │ payload.data: {...}      │
                     │ sequence: 2     │      │ status: ok               │
                     └─────────────────┘      │ metrics: {...}           │
                                              └──────────────────────────┘
```
