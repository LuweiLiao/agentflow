## 6. 验收演示场景

**场景名称：** 四旋翼无人机 PID 控制器全流程

```
[需求分析]    分析目标、约束
    ↓
[系统建模]    动力学模型 → 传递函数
    ↓
[控制器设计]  Ziegler-Nichols → PID 参数
    ↓
[C++代码生成]  ArduPilot 框架 → PID.cpp
    ↓
[编译测试]     scons → 单元测试
    ↓
[输出报告]     综合结果 → 交付文档
```

**执行流程：**
1. 在欢迎页输入「为四旋翼无人机设计PID控制器并生成C++代码，然后编译测试」
2. 点击 AI 自动生成 → 编排 Agent 拆解为 6 个节点 + 分配 Profile
3. CSS Grid 画布展示 6 个 Agent 卡片 + 输出卡片
4. 点击运行 → AgentRunner 依次执行每个节点
5. 节点颜色灰→蓝→绿变化，输出卡片显示每一步结果
6. 完成弹窗显示 cost / duration / turns / model

## 7. 风险 & 应对 — 实际处理

| 风险 | 实际应对 |
|---|---|
| LLM API Key 管理 | 环境变量硬编码 + Multi-Provider 抽象 |
| 状态推送 (WebSocket) | 简化为 REST + 前端 setTimeout 模拟 |
| 画布性能问题 | CSS Grid 轻量渲染，无 React 依赖 |
| Orchestrator 状态持久化 | 内存状态，无数据库依赖 |
| React Flow 集成复杂度 | 改用原生 CSS Grid，大幅降低前端复杂度 |
