**SDK 使用示例：**

```python
from agentflow import Workflow, node

@node(input_schema={"text": "string"}, output_schema={"summary": "string"})
def summarize(text: str) -> dict:
    # 调用 LLM
    return {"summary": llm_call(f"Summarize: {text}")}

wf = Workflow("my_workflow")
wf.add_node(summarize)
wf.set_input("text", "Long article content...")
result = wf.run()
print(result["summary"])
```

## 5. 验收标准

| # | 验收条件 | 判定方式 |
|---|---|---|
| 1 | Python SDK 可通过 `pip install agentflow` 安装并成功执行工作流 | 端到端测试 |
| 2 | 节点市场可浏览、搜索、安装第三方节点 | 人工测试 |

| 5 | RBAC：Editor 不能管理用户，Admin 可以 | 权限测试 |
| 6 | `docker compose up -d` 后系统在 60 秒内可访问 | 计时测试 |
| 7 | Webhook 触发：POST 到 Webhook URL → 启动工作流 → 返回结果 | 集成测试 |
| 8 | Cron 触发：定时执行工作流，结果写入执行历史 | 集成测试 |
| 9 | 代码覆盖率 > 75% | CI 报告 |
| 10 | 无 P0/P1 Bug | Bug 追踪系统 |
| 11 | 公测文档覆盖：快速开始、API 参考、部署指南 | 文档评审 |

## 7. 风险 & 应对

| 风险 | 应对 |
|---|---|
| SDK API 设计不合理导致后续断裂 | Phase 4 前进行 SDK 设计 review + 语义化版本保证 |
| 节点市场沙箱逃逸 | 使用 gVisor 隔离 + 资源限制 (CPU/Memory/Network) |
| Docker Compose 在生产环境不够 | Phase 5 提供 Kubernetes Helm Chart |
| 公测用户涌入导致不稳定 | 后端限流 + 队列缓冲 + 熔断降级 |
| 开源 / 闭源决策未定 | 暂定闭源公测，根据反馈决定开源策略 |
