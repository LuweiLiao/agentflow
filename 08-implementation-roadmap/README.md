## 分阶段规划表

| 阶段 | 时间 | 核心目标 | 关键交付物 | 人员需求 |
|---|---|---|---|---|
| **Phase 1** 方案定稿 | Week 1-2 | 冻结数据契约，完成 UI 交互设计 | Spec 文档、Figma 原型、数据契约 | 1 PM + 1 架构师 + 1 设计师 |
| **Phase 2** MVP 实现 | Week 3-8 | 端到端跑通最小闭环 | Compiler Agent、3 模板、Orchestrator、画布 | 2 全栈 + 1 ML Engineer |
| **Phase 3** 生产增强 | Week 9-12 | 提升可用性、可靠性与安全性 | KMS 集成、并行执行、执行历史、模板管理 | 2 后端 + 1 前端 |
| **Phase 4** 生态开放 | Week 13-16 | 开放生态，降低接入门槛 | SDK、节点市场、RBAC、Docker 部署、源编排 | 2 后端 + 1 全栈 + 1 DevOps |

## 里程碑与检查点

| 里程碑 | 时间 | 验收标准 | 是否硬截止 |
|---|---|---|---|

## 依赖关系

```
Phase 1 ──▶ Phase 2 ──▶ Phase 3 ──▶ Phase 4
   │            │            │
   ├▶ 数据契约   └▶ Compiler  └▶ KMS 集成
   │               │            │
   └▶ UI 设计      ├▶ 模板      └▶ 模板管理 UI
                   │
                   ├▶ Orchestrator
                   │
                   └▶ 画布 (需 UI 设计稿)
```

**关键路径：** 数据契约 → Compiler Agent → Orchestrator → 端到端跑通 → 生产增强

---

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| 数据契约反复修改 | 中 | 高 — 导致各模块重新实现 | Phase 1 充分评审，增加 freeze 机制 |
| Compiler Agent 复杂度超预期 | 中 | 中 — MVP 延迟 | Phase 2 先做最小子集，后续迭代 |
| React Flow 自定义节点性能 | 低 | 中 — 大画布卡顿 | 使用虚拟化 + 节点懒加载 |
| Orchestrator 状态一致性 | 中 | 高 — 执行结果不可靠 | 引入事件溯源 + 幂等执行 |
| 团队人手不足 | 高 | 中 — 进度落后 | 优先保证关键路径，裁剪非核心功能 |
