# 06.2 — 运行时安全架构

> 文档版本：v0.2 | 最后更新：2026-06-10

## 概述

AgentFlow 运行时当前处于 MVP 阶段，安全架构以 **快速迭代** 优先。生产级安全措施将在 Phase 3 引入。

## 当前安全措施

### 1. API 密钥管理

| 措施 | 说明 |
|------|------|
| 环境变量注入 | API Key 通过环境变量传递，不在代码中硬编码 |
| 进程隔离 | AgentRunner 在独立 Python 进程中运行 |
| 最小依赖 | 无第三方密钥管理服务依赖 |

**风险：** `start-agentflow.sh` 中硬编码了 ZAI_API_KEY，建议迁移到 `systemd EnvironmentFile`。

### 2. 执行沙箱

当前 Agent 运行在 **无沙箱环境**：

- Agent 使用 `subprocess.run(cmd, shell=True)` 执行 Shell 命令
- 可以访问文件系统、网络、系统调用
- 无容器/VM 隔离

**缓解措施：**
- AgentRunner 的工作目录是临时目录 (`tempfile.mkdtemp`)
- 执行完成后清理临时目录 (`shutil.rmtree`)
- 每个 Agent 节点的工作目录独立隔离

### 3. 超时控制

| 层级 | 超时 | 说明 |
|------|------|------|
| LLM 调用 | 120s | 单个 API 请求最大等待时间 |
| Agent 节点 | 模板配置 (默认 120-300s) | 单节点总执行时间 |
| 子进程 | 30s | 单条 Shell 命令最大执行时间 |
| 前端 fetch | 300s | 前端 AbortController 超时 |

## Phase 3 计划引入的安全措施

### 1. Docker 容器隔离

```yaml
Agent Runner:
  isolation: docker
  image: agentflow-runner:latest
  read_only_rootfs: true
  capabilities: [NET_RAW, NET_ADMIN]  # 按需
  security_opt:
    - no-new-privileges:true
    - apparmor:agentflow-default
  resource_limits:
    cpus: 2.0
    memory: 4G
    pids: 50
```

### 2. 密钥管理 (KMS)

```python
# Phase 3: 使用 KMS 替代环境变量
key = kms.decrypt("agentflow/provider/zhipu/key")
```

### 3. 审计日志

```json
{
  "event": "agent_execution",
  "node_id": "a1",
  "timestamp": "...",
  "command_count": 5,
  "files_read": ["/workspace/src/main.py"],
  "files_written": ["/workspace/output/report.md"]
}
```

## 安全设计原则

| 原则 | MVP 阶段 | Phase 3+ |
|------|---------|----------|
| 最小权限 | API Key 只读 | 容器只读文件系统 + 网络白名单 |
| 默认安全 | 临时目录隔离 | 容器隔离 + seccomp |
| 纵深防御 | 单层（超时） | 多层（网络/文件/系统调用过滤） |
| 可审计 | stderr 日志 | 结构化审计日志 |
