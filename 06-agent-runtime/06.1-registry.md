# 06.1 — Agent 类型注册表

Agent 类型注册表（Agent Registry）是 AgentFlow 中所有 Agent 类型的中央元数据存储。每个 Agent 类型定义了其实例化所需的一切：镜像、超时、重试策略、系统提示词、可用工具集及成本系数。

---

## 注册表格式

注册表由一个或多个 YAML 文件组成，运行时在启动时合并加载。默认位置：`config/agent-registry.yaml`。

### 完整结构

```yaml
# 注册表顶层是一个 map，key 为 Agent 类型标识符（snake_case）
<agent_type_id>:
  description: "<人类可读的描述>"
  icon: "<emoji 或图标名称>"
  category: "<分类标签>"
  base_image: "<容器镜像名称:tag>"
  default_timeout_s: <整数，最大执行秒数>
  default_retry: <整数，失败重试次数>
  system_prompt: |
    <系统级提示词，定义 Agent 的行为准则>
  allowed_tools:
    - "<工具名称>"
    - "<工具名称>"
  cost_multiplier: <浮点数，成本乘数>
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `description` | string | 是 | Agent 类型的人类可读描述，用于 UI 展示 |
| `icon` | string | 是 | 图标标识（Emoji 或 Material Icon 名称） |
| `category` | string | 是 | 分类标签，用于组织与筛选 |
| `base_image` | string | 是 | Docker 镜像名称及标签，必须可从运行时节点拉取 |
| `default_timeout_s` | integer | 是 | 单次执行的最大时长（秒），超时强制终止 |
| `default_retry` | integer | 是 | 失败后自动重试次数（0 = 不重试） |
| `system_prompt` | string | 否 | Agent 系统提示词，定义角色、行为和约束 |
| `allowed_tools` | string[] | 否 | Agent 可调用的工具白名单 |
| `cost_multiplier` | float | 否 | 成本系数，用于计费与配额计算（默认 1.0） |

## 预置 Agent 类型

以下为 AgentFlow 内置的预置 Agent 类型：

### `algorithm-designer` — 算法设计 Agent

```yaml
algorithm-designer:
  description: "算法设计Agent — 负责算法设计、伪代码生成和复杂度分析"
  icon: "🧮"
  category: "design"
  base_image: "agentflow-agent:latest"
  default_timeout_s: 120
  default_retry: 2
  system_prompt: |
    你是一位资深的算法设计师。你的职责包括：
    - 根据需求设计高效算法
    - 编写清晰、可读的伪代码
    - 进行时间和空间复杂度分析
    - 提供算法正确性的非正式证明

    请遵循以下原则：
    1. 优先选择已知的最优算法
    2. 给出算法步骤的详细解释
    3. 分析边界情况和输入约束
    4. 如果输入不明确，主动提问澄清
  allowed_tools:
    - python_repl
  cost_multiplier: 1.0
```

compile-test:

## 注册表管理

### 动态注册

运行时启动后可通过 API 动态注册新的 Agent 类型：

```bash
POST /api/v1/registry/register
Content-Type: application/json

{
  "type_id": "data-analyzer",
  "description": "数据分析Agent — 负责数据清洗、统计分析和可视化",
  "icon": "📊",
  "category": "analysis",
  "base_image": "agentflow-agent:latest",
  "default_timeout_s": 240,
  "default_retry": 1,
  "system_prompt": "...",
  "allowed_tools": ["python_repl", "file_system"],
  "cost_multiplier": 1.2
}
```

### 热加载

注册表支持热加载（不重启运行时）：

1. 修改 `config/agent-registry.yaml`
2. 向运行时发送 `SIGHUP` 信号或调用 `POST /api/v1/registry/reload`
3. 运行时验证新配置的合法性，增量更新注册表

### 验证规则

注册表加载时会自动执行以下验证：

| 规则 | 说明 |
|------|------|
| 类型 ID 唯一性 | 不允许重复的 `type_id` |
| 镜像可达性 | `base_image` 必须在运行时节点中存在或可拉取 |
| 超时范围 | `default_timeout_s` 必须在 10–3600 之间 |
| 重试上限 | `default_retry` 不得超过 5 |
| 工具存在性 | `allowed_tools` 中的所有工具必须在工具注册表中存在 |
| 成本范围 | `cost_multiplier` 必须在 0.1–10.0 之间 |

    default_retry: int

class AgentRegistry:

                raise UnknownAgentTypeError(type_id)
