# 06.1 — Provider 注册与模型映射

> 文档版本：v0.2 | 最后更新：2026-06-10

## 概述

AgentRunner 通过 **静态配置表** 和 **模型名前缀匹配** 实现 Provider 的自动发现和注册。无需运行时注册，只需设置环境变量即可使用。

## Provider 配置表

每个 Provider 在 `agent_runner.py:PROVIDER_CONFIGS` 中注册为一条配置：

```python
PROVIDER_CONFIGS = {
    "deepseek": {
        "key_env": "DEEPSEEK_API_KEY",    # 环境变量名
        "base_env": "DEEPSEEK_BASE_URL",   # 可选的 Base URL 覆盖
        "default_base": "https://api.deepseek.com/v1"  # 默认 Base URL
    },
    "zhipu": {
        "key_env": "ZAI_API_KEY",
        "base_env": "ZAI_BASE_URL",
        "default_base": "https://open.bigmodel.cn/api/paas/v4"
    },
    # ... 共 12 个 Provider
}
```

## 模型→Provider 映射

通过 **精确匹配 + 前缀匹配** 自动确定模型对应的 Provider：

```python
MODEL_PROVIDER_MAP = {
    "deepseek-chat": "deepseek",
    "glm-5-turbo": "zhipu",
    "grok-3": "xai",
    "gpt-4o": "openai",
    # ... 精确匹配表
}

# 前缀匹配规则（精确匹配失败时）
PREFIX_RULES = [
    ("deepseek", "deepseek"),
    ("glm-", "zhipu"),
    ("grok", "xai"),
    ("gpt-", "openai"),
    ("qwen", "aliyun"),
    ("ernie", "baidu"),
    # ...
]
```

### 匹配流程

```
模型名 (如 "deepseek-chat")
    │
    ▼
精确匹配 MODEL_PROVIDER_MAP?
    Yes → 返回对应 Provider
    No  → 继续
    │
    ▼
前缀匹配 PREFIX_RULES?
    Yes → 返回对应 Provider
    No  → 使用 OpenAI Provider (通用 fallback)
```

## API Key 发现链

```
指定 Provider 的环境变量 (如 DEEPSEEK_API_KEY)
    │
    ▼
未设置？→ ZHIPU fallback (仅 zhipu provider)
    │
    ▼
仍未设置？→ OPENAI_API_KEY (仅限 deepseek/openai/siliconflow)
```

## 运行时配置

通过环境变量 `AGENT_MODEL` 设定默认模型：

```bash
export AGENT_MODEL="deepseek-chat"   # 使用 DeepSeek
export AGENT_MODEL="glm-5-turbo"     # 使用 智谱 GLM
export AGENT_MODEL="gpt-4o"          # 使用 OpenAI
```

未设置时默认：`deepseek-chat`。

## 扩展新 Provider

添加新 Provider 只需三步：

1. 在 `PROVIDER_CONFIGS` 中添加配置条目
2. 在 `MODEL_PROVIDER_MAP` 中添加模型映射
3. 设置对应的环境变量

无需重启注册表或修改注册流程。
