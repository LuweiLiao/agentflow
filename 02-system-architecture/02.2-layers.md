# 02.2 6+1 层级架构

> 本文档详细描述 AgentFlow 系统的 6+1 层级划分、各层职责、接口边界及演进路线。

---

## 层级总览

AgentFlow 采用 **6+1 层级架构**设计，以**源编排层（L0）** 和**核心引擎层（L1）** 为基石，向上逐步扩展 SDK（L2）、市场（L3）、画布 UI（L4）、团队版（L5）和运维（L6）能力。

```
 ┌──────────────────────────────────────────────────┐
 │                   L6 运维 (Defer)                  │
 │  部署 / 监控 / 日志 / 运维控制台                    │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │               L5 团队版 (Defer)                    │
 │  团队协作 / 权限管理 / 共享工作区                   │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │            L4 画布 UI (复用 🟦)                    │
 │      React Flow 可视化工作流编辑器                  │
 │     节点拖拽 / 连线 / 配置面板 / 执行状态展示        │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │               L3 市场 (Defer)                      │
 │        Agent 模板市场 / 节点市场                    │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │               L2 SDK (Defer)                       │
 │   对外编程接口 / 插件 SDK / 自定义节点开发框架       │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │         L1 核心引擎 (自研 🟥)                       │
 │  Compiler + Orchestrator + 模板系统                │
 │  —— 工作流编译、编排调度、运行时代理                 │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │        L0 源编排 (自研 🟥) ───── 基石               │
 │  编排元模型 / 节点类型注册 / 画布状态管理 / Schema  │
 └──────────────────────────────────────────────────┘
```

## L0 — 源编排层（自研 🟥）

### 职责

L0 是整个 AgentFlow 的基石层，定义了工作流编排的核心元模型和数据契约。所有上层能力都建立在 L0 提供的抽象之上。

### 关键组件

| 组件 | 说明 |
|------|------|
| **编排元模型** | 定义 `WorkflowJSON`、`Node`、`Edge`、`Port`、`DataMap` 等核心数据结构 |
| **节点类型注册表** | 集中管理所有节点类型（Agent 节点、工具节点、条件节点、输入/输出节点等），支持热注册 |
| **画布状态管理** | 维护画布运行时状态（节点状态、数据流、执行进度），提供状态驱动的画布更新 API |
| **Schema 契约库** | 定义 5 阶段之间所有 Schema 契约（WorkflowJSON、PromptTask、DockerRunSpec、Envelope、NodeResult） |
| **事件总线** | 定义工作流生命周期事件（node_started / node_completed / node_failed / workflow_completed 等） |

### 接口

```
// 节点类型注册
registerNodeType(type: string, definition: NodeTypeDefinition): void

// 工作流序列化
serializeWorkflow(nodes: Node[], edges: Edge[]): WorkflowJSON

// 状态查询
getNodeState(workflowId: string, nodeId: string): NodeExecutionState

// 画布更新
updateCanvas(workflowId: string, updates: CanvasUpdate[]): void
```

### 演进状态

**已完成**（核心元模型和节点类型注册已实现），持续迭代 Schema 定义。

---

## L1 — 核心引擎层（自研 🟥）

### 职责

L1 是 AgentFlow 的执行核心，负责将 L0 的元模型实例化为可运行的工作流，包括编译、编排、调度和执行。

### 关键组件

| 组件 | 说明 |
|------|------|
| **Prompt Compiler** | 将 WorkflowJSON 编译为 PromptTask[]，包含拓扑排序、模板展开、变量注入 |
| **Orchestrator** | 基于 Hermes Agent，接收 PromptTask[]，生成 Docker 运行指令，管理任务生命周期 |
| **模板系统** | 管理系统提示模板（System Prompt）、用户提示模板（User Prompt）、输出 Schema 模板 |
| **结果汇聚引擎** | 收集 Envelope JSON，执行 Schema 验证、数据提取、错误处理，触发下一节点 |
| **运行时上下文** | 维护工作流执行期间的变量上下文，支持跨节点数据传递 |

### 架构关系

```
                    ┌──────────┐
                    │ L0 元模型 │
                    └────┬─────┘
                         │ 实例化
                    ┌────▼─────┐
           ┌────────┤ L1 核心引擎├────────┐
           │        └──────────┘        │
     ┌─────▼─────┐              ┌──────▼──────┐
     │ Compiler   │              │ Orchestrator│
     │ (Claude    │              │ (Hermes     │
     │  Code)     │              │  Agent)     │
     └─────┬─────┘              └──────┬──────┘
           │                           │
     ┌─────▼─────┐              ┌──────▼──────┐
     │ 模板系统   │              │ Docker 集群  │
     └───────────┘              └─────────────┘
```

### 演进状态

**开发中**，Compiler 和 Orchestrator 为当前主要开发目标。

---

## L2 — SDK 层（Defer）

### 职责

为开发者提供编程接口，允许通过代码（而非画布）创建和管理 AgentFlow 工作流，以及开发自定义节点和插件。

### 计划内容

| 组件 | 说明 |
|------|------|
| **AgentFlow SDK** | 支持 Python/TypeScript 两种语言，提供 `WorkflowBuilder`、`NodeConfig`、`executeWorkflow` 等 API |
| **插件框架** | 定义插件生命周期（init / before_step / after_step / destroy），支持钩子注入 |
| **自定义节点 SDK** | 允许开发者实现自定义节点类型，注册到 L0 的节点类型注册表中 |

### 示例（设计中的 API）

```python
from agentflow import Workflow, AgentNode, ToolNode

wf = Workflow("code-review-pipeline")
review = AgentNode("code-review", model="claude-sonnet-4", system_prompt_ref="tpl://code-review")
read_file = ToolNode("read-source", tool="file_reader", path="./src")
wf.connect(read_file, review, data_map={"content": "input"})
wf.run()
```

### 预期里程碑

- Phase 1: Python SDK 核心 API
- Phase 2: TypeScript SDK + 类型定义
- Phase 3: 插件框架 + 自定义节点 SDK

## L3 — 市场层（Defer）

### 职责

提供 Agent 模板和节点的共享市场，用户可发布、搜索、安装社区贡献的模板和节点。

### 计划内容

| 组件 | 说明 |
|------|------|
| **模板市场** | 托管 Prompt 模板和完整工作流模板，支持版本管理 |
| **节点市场** | 托管自定义节点实现（通过 L2 SDK 开发），支持一键安装 |
| **评分与评价** | 用户对模板和节点进行评分、评论 |

### 预期里程碑

- Phase 1: 模板市场 CLI 搜索/安装
- Phase 2: 画布内可视化模板浏览和导入
- Phase 3: 节点市场 + 社区发布流程

---

## L4 — 画布 UI 层（复用 🟦）

### 职责

用户使用 AgentFlow 的主要交互界面，基于 **React Flow** 构建的可视化工作流编辑器。

### 关键能力

| 能力 | 说明 |
|------|------|
| **可视化编辑** | 节点拖拽、连线、删除、布局调整 |
| **配置面板** | 编辑节点参数（模型选择、System Prompt、工具配置等） |
| **执行状态展示** | 实时显示节点运行状态（pending/running/success/failed） |
| **执行回放** | 动画展示数据流在工作流中的传递路径 |
| **结果预览** | 点击节点查看详细执行结果和产物 |

### 复用策略

React Flow 作为 OSS 组件直接复用（详见 [02.3-oss-strategy.md](./02.3-oss-strategy.md)），AgentFlow 在其基础上封装了业务组件。

### 架构关系

```
┌──────────────────────────────────────────────┐
│              L4 画布 UI                        │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │         React Flow Core               │    │ ← OSS 复用
│  └──────────────────────────────────────┘    │
│                    │                         │
│  ┌─────────────────┴──────────────────┐     │
│  │    AgentFlow 自定义组件层            │     │ ← 业务封装
│  │  AgentNode / ToolNode / ConditionNode │    │
│  │  配置面板 / 执行状态条 / 结果展示    │     │
│  └────────────────────────────────────┘     │
│                    │                         │
│  ┌─────────────────┴──────────────────┐     │
│  │     API 客户端层（连接 L0/L1）       │     │
│  │  提交 WorkflowJSON / 轮询状态       │     │
│  └────────────────────────────────────┘     │
└──────────────────────────────────────────────┘
```

## L5 — 团队版（Defer）

### 职责

为团队协作场景提供多用户支持，包括权限管理、共享工作区、版本控制。

### 计划内容

| 组件 | 说明 |
|------|------|
| **用户与组织管理** | 用户注册、组织创建、角色分配 |
| **权限系统** | 工作流级/模板级/节点级的读写权限 |
| **共享工作区** | 团队共享的工作流模板和运行历史 |
| **版本历史** | 工作流变更的版本控制与回滚 |
| **注释与评论** | 在画布节点上添加注释，支持团队讨论 |

### 预期里程碑

- Phase 1: 基础用户系统 + 个人工作区
- Phase 2: 组织管理 + 共享工作区
- Phase 3: 权限系统 + 版本历史

---

## L6 — 运维层（Defer）

### 职责

提供生产环境所需的部署、监控、日志和运维管理能力。

### 计划内容

| 组件 | 说明 |
|------|------|
| **部署管理** | 一键部署脚本（Docker Compose / Kubernetes），环境配置管理 |
| **运行监控** | 仪表盘展示工作流执行量、成功率、平均耗时、资源使用率 |
| **日志聚合** | 集中收集所有 Agent 容器日志，支持全文检索 |
| **告警系统** | 异常检测（执行失败率突增、Token 消耗异常等）并发送通知 |
| **运维控制台** | Web 界面查看运行时状态、管理节点、配置告警规则 |

### 预期里程碑

- Phase 1: Docker Compose 部署 + 基础日志收集
- Phase 2: Kubernetes Helm Chart + 监控仪表盘
- Phase 3: 完整运维控制台

## 层间接口与依赖关系

```
┌──────────────────────────────────────────────────┐
│  L6 运维 ── 依赖: L0~L5 的运行时数据              │
│  接口: 日志流 / 指标 API / 健康检查 API           │
├──────────────────────────────────────────────────┤
│  L5 团队版 ── 依赖: L4 画布 UI, L0 元模型        │
│  接口: 用户认证 / 权限校验                        │
├──────────────────────────────────────────────────┤
│  L4 画布 UI ── 依赖: L0 Schema, L1 API           │
│  接口: WorkflowJSON 提交 / 状态轮询               │
├──────────────────────────────────────────────────┤
│  L3 市场 ── 依赖: L2 SDK, L1 模板系统            │
│  接口: 模板/节点 发布/安装 API                    │
├──────────────────────────────────────────────────┤
│  L2 SDK ── 依赖: L0 元模型, L1 核心引擎          │
│  接口: WorkflowBuilder / executeWorkflow          │
├──────────────────────────────────────────────────┤
│  L1 核心引擎 ── 依赖: L0 元模型和 Schema          │
│  接口: compile / orchestrate / aggregate          │
├──────────────────────────────────────────────────┤
│  L0 源编排 ── 无下层依赖                          │
│  接口: 元模型定义 / 节点注册表 API                │
└──────────────────────────────────────────────────┘
```

## 演进路线图

```
时间 ────────────────────────────────────────────►

MVP (当前)         v0.2           v0.5            v1.0
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────────┐
│ L0 源编排  │   │ L1 核心  │   │ L4 画布  │   │ L2 SDK       │
│ (基础)    │   │ 引擎    │   │ UI 完善  │   │ L3 市场      │
│ L1 核心   │   │ (完整)  │   │          │   │ L5 团队版    │
│ 引擎(MVP) │   │          │   │          │   │ L6 运维      │
│ L4 画布   │   │          │   │          │   │              │
│ UI(原型)  │   │          │   │          │   │              │
└──────────┘   └──────────┘   └──────────┘   └──────────────┘

                       ▲              ▲               ▲
                   核心可用       体验完善       全功能生产就绪
```

- **当前阶段（MVP）**：聚焦 L0 + L1 + L4，打通 5 阶段核心流水线，实现端到端的最小可用工作流执行
- **v0.2**：L1 核心引擎功能完整（Compiler 支持完整模板语法、Orchestrator 支持并行调度和重试）
- **v0.5**：L4 画布 UI 体验完善（执行回放、结果预览、节点配置面板）
- **v1.0**：L2~L6 全面就绪，进入生产可用阶段
