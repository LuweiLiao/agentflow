# 02.2 6+1 层级架构

> 本文档详细描述 AgentFlow 系统的 6+1 层级划分、各层职责、接口边界及演进路线。

---

## 层级总览

AgentFlow 采用 **6+1 层级架构**设计，以**源编排层（L0）** 和**核心引擎层（L1）** 为基石，向上逐步扩展 SDK（L2）、市场（L3）、画布 UI（L4）、团队版（L5）和运维（L6）能力。

```
 ┌──────────────────────────────────────────────────┐
 │               L6 运维 (Defer)                      │
 │  部署 / 监控 / 日志 / 运维控制台                    │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │             L5 团队版 (Defer)                      │
 │  团队协作 / 权限管理 / 共享工作区                   │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │          L4 画布 UI (自研 🟥)                      │
 │    CSS Grid HTML 可视化工作流编辑器                 │
 │   节点卡片 / 执行状态 / 配置面板                    │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │             L3 市场 (Defer)                        │
 │      Agent 模板市场 / 节点市场                      │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │             L2 SDK (Defer)                         │
 │   对外编程接口 / 插件 SDK / 自定义节点开发框架       │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │     L1 核心引擎 (自研 🟥)                           │
 │  Compiler + AgentRunner + 模板系统                 │
 │  + 多 Provider 抽象层 + 结果汇聚引擎               │
 │  —— 工作流编译、DAG 并行执行、多供应商调度          │
 └──────────────────────┬───────────────────────────┘
                        │
 ┌──────────────────────┴───────────────────────────┐
 │     L0 源编排 (自研 🟥) ───── 基石                  │
 │  编排元模型 / 节点类型注册 / GLM-5-Turbo 需求拆解   │
 │  画布状态管理 / 数据契约                            │
 └──────────────────────────────────────────────────┘
```

## L0 — 源编排层（自研 🟥）

### 职责

L0 是整个 AgentFlow 的基石层，定义了工作流编排的核心元模型和数据契约，并负责通过 GLM-5-Turbo 自动将用户一句话需求拆解为工作流。所有上层能力都建立在 L0 提供的抽象之上。

### 关键组件

| 组件 | 说明 |
|------|------|
| **编排元模型** | 定义 `NodeDef`、`EdgeDef`、`WorkflowJSON`、`PromptTask`、`EnvelopeJSON` 等核心数据结构 |
| **GLM-5-Turbo 需求拆解** | 调用智谱 GLM-5-Turbo API，将用户需求自动拆解为 N 个 Agent 节点，每个节点含 profile 类型 |
| **本地模板降级** | 当 API 不可用时，使用内置模板（PID控制器、企业网站、机器学习、自动驾驶），支持数量裁剪 |
| **节点类型注册表** | 管理 6 种 Profile 类型（analysis/design/dev/test/doc/deploy），支持 profile 推断 |
| **画布状态管理** | 维护画布运行时状态（节点状态、数据流、执行进度），提供状态驱动的画布更新 API |
| **数据契约库** | 定义 6 阶段之间所有数据契约（WorkflowJSON、PromptTask、EnvelopeJSON、NodeResult） |

### 接口

```
# 源编排 — 需求拆解
POST /api/decompose {requirement, count} → {nodes, edges, count}

# 节点类型推断
inferProfile(node) → "analysis" | "design" | "dev" | "test" | "doc" | "deploy"

# 工作流序列化
WorkflowJSON.from_api_request(nodes, requirement, edges) → WorkflowJSON

# 状态查询
getNodeState(workflowId, nodeId) → NodeExecutionState
```

### 演进状态

**已完成** — 核心元模型、GLM-5-Turbo 集成、本地模板降级均已实现，持续迭代拆解质量。

## L1 — 核心引擎层（自研 🟥）

### 职责

L1 是 AgentFlow 的执行核心，负责将 L0 的工作流定义实例化为可运行的任务，包括编译、DAG 并行执行、多 Provider 调度和结果汇聚。

### 关键组件

| 组件 | 说明 |
|------|------|
| **Prompt Compiler** | 将 WorkflowJSON 编译为 PromptTask[]，包含 Kahn 拓扑排序、YAML 模板加载、变量渲染、上游上下文注入 |
| **AgentRunner** | 通用 Agent 执行器，支持任何 OpenAI 兼容 API，内置 12+ 供应商配置，自动根据模型名选择 provider |
| **DAG 并行执行引擎** | 按拓扑深度分组，同组节点通过 ThreadPoolExecutor 并行执行，组间串行推进 |
| **多 Provider 抽象层** | 统一管理 12 个 LLM API 供应商（DeepSeek、智谱 GLM、xAI Grok、OpenAI、阿里通义、月之暗面等） |
| **模板系统** | 管理系统提示模板（System Prompt）、用户提示模板（User Prompt），每个 profile 独立 YAML 文件 |
| **结果汇聚引擎** | 收集 Envelope JSON，提取结果摘要和指标，更新节点状态，计算总成本/耗时 |
| **校验器** | `validate_workflow()` 进行基本完整性检查（节点引用、profile 合法性） |

### 架构关系

```
                    ┌──────────┐
                    │ L0 元模型 │
                    └────┬─────┘
                         │ 实例化
                    ┌────▼─────┐
           ┌────────┤ L1 核心引擎├──────────┐
           │        └──────────┘          │
     ┌─────▼─────┐              ┌────────▼──────┐
     │ Compiler   │              │ DAG 执行引擎   │
     │ (自研)     │              │ (ThreadPool)   │
     │ YAML 模板  │              │ ┌────────────┐ │
     │ 拓扑排序   │              │ │ AgentRunner │ │
     └─────┬─────┘              │ │ 12+Provider │ │
           │                    │ └────────────┘ │
           │                    └────────┬───────┘
     ┌─────▼─────┐                      │
     │ 模板系统   │              ┌───────▼───────┐
     │ YAML 文件  │              │ 结果汇聚引擎   │
     └───────────┘              └───────────────┘
```

### 演进状态

**已完成** — Compiler、AgentRunner、DAG 并行执行引擎、结果汇聚均已实现并集成。

## L2 — SDK 层（Defer）

### 职责

为开发者提供编程接口，允许通过代码（而非画布）创建和管理 AgentFlow 工作流，以及开发自定义节点和插件。

### 计划内容

| 组件 | 说明 |
|------|------|
| **AgentFlow SDK** | 支持 Python/TypeScript 两种语言，提供 `WorkflowBuilder`、`NodeConfig`、`executeWorkflow` 等 API |
| **插件框架** | 定义插件生命周期（init / before_step / after_step / destroy），支持钩子注入 |
| **自定义节点 SDK** | 允许开发者实现自定义节点类型，注册到 L0 的节点类型注册表中 |

### 示例（设计中的 API）

```python
from agentflow import Workflow, AgentNode

wf = Workflow("code-review-pipeline")
review = AgentNode("code-review", profile="analysis", model="gpt-4o")
wf.connect(review, next_node)
wf.run()
```

### 预期里程碑

- Phase 1: Python SDK 核心 API
- Phase 2: TypeScript SDK + 类型定义
- Phase 3: 插件框架 + 自定义节点 SDK

## L3 — 市场层（Defer）

### 职责

提供 Agent 模板和节点的共享市场，用户可发布、搜索、安装社区贡献的模板和节点。

### 计划内容

| 组件 | 说明 |
|------|------|
| **模板市场** | 托管 Prompt 模板和完整工作流模板，支持版本管理 |
| **节点市场** | 托管自定义节点实现（通过 L2 SDK 开发），支持一键安装 |
| **评分与评价** | 用户对模板和节点进行评分、评论 |

### 预期里程碑

- Phase 1: 模板市场 CLI 搜索/安装
- Phase 2: 画布内可视化模板浏览和导入
- Phase 3: 节点市场 + 社区发布流程

## L4 — 画布 UI 层（自研 🟥）

### 职责

用户使用 AgentFlow 的主要交互界面，基于 **CSS Grid HTML**（纯前端，无第三方 UI 框架依赖）构建的可视化工作流编辑器。

### 关键能力

| 能力 | 说明 |
|------|------|
| **可视化展示** | CSS Grid 网格布局展示 Agent 节点卡片、输出卡片、状态标识 |
| **一步生成** | 输入需求 → 自动编排 → CSS Grid 渲染全流程节点 |
| **执行状态实时显示** | 通过 CSS class 动态切换展示节点运行状态（pending/executing/done） |
| **节点配置面板** | 右侧面板展示选中节点的详细配置和日志 |
| **结果预览** | 点击节点查看详细执行结果、耗时、Token 消耗、文件产物 |

### 架构关系

```
┌──────────────────────────────────────────────┐
│              L4 画布 UI                        │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │         CSS Grid 布局引擎              │    │ ← 自研：纯 HTML/CSS
│  │  grid-template-columns: 4列           │    │
│  │  grid-auto-rows: 自适应行高            │    │
│  └──────────────────────────────────────┘    │
│                    │                         │
│  ┌─────────────────┴──────────────────┐     │
│  │    AgentFlow 自定义组件层            │     │ ← 自研：业务封装
│  │  c-agent / c-output / c-arrow       │    │
│  │  sbadge / c-vert-line 等            │     │
│  └────────────────────────────────────┘     │
│                    │                         │
│  ┌─────────────────┴──────────────────┐     │
│  │     API 客户端层（连接 L0/L1）       │     │
│  │  POST /api/decompose                │     │
│  │  POST /api/execute                  │     │
│  └────────────────────────────────────┘     │
└──────────────────────────────────────────────┘
```

### 画布渲染结构

每个 Agent 节点在 CSS Grid 中占 4 列：
1. **步骤号**（左对齐）：第 N 步
2. **Agent 卡片**：图标 + 名称 + 状态标识
3. **箭头**：→ 表示数据流向
4. **输出卡片**：执行结果摘要

## L5 — 团队版（Defer）

### 职责

为团队协作场景提供多用户支持，包括权限管理、共享工作区、版本控制。

### 计划内容

| 组件 | 说明 |
|------|------|
| **用户与组织管理** | 用户注册、组织创建、角色分配 |
| **权限系统** | 工作流级/模板级/节点级的读写权限 |
| **共享工作区** | 团队共享的工作流模板和运行历史 |
| **版本历史** | 工作流变更的版本控制与回滚 |
| **注释与评论** | 在画布节点上添加注释，支持团队讨论 |

### 预期里程碑

- Phase 1: 基础用户系统 + 个人工作区
- Phase 2: 组织管理 + 共享工作区
- Phase 3: 权限系统 + 版本历史

---

## L6 — 运维层（Defer）

### 职责

提供生产环境所需的部署、监控、日志和运维管理能力。

### 计划内容

| 组件 | 说明 |
|------|------|
| **部署管理** | 一键部署脚本（Docker Compose / systemd），环境配置管理 |
| **运行监控** | 仪表盘展示工作流执行量、成功率、平均耗时、Token 消耗 |
| **日志聚合** | 集中收集所有 Agent 执行日志，支持全文检索 |
| **告警系统** | 异常检测（执行失败率突增、成本异常等）并发送通知 |
| **运维控制台** | Web 界面查看运行时状态、管理 API Key、配置告警规则 |

### 预期里程碑

- Phase 1: systemd 服务 + 基础日志收集
- Phase 2: Docker Compose 部署 + 监控仪表盘
- Phase 3: 完整运维控制台

## 演进路线图

```
时间 ────────────────────────────────────────────►

MVP (当前)               v0.2              v0.5              v1.0
┌──────────────────┐   ┌──────────────┐   ┌──────────┐   ┌──────────────┐
│ L0 源编排 (基础)   │   │ L1 完整     │   │ L4 画布  │   │ L2 SDK       │
│ L1 核心引擎 (MVP)  │   │ 引擎       │   │ UI 完善  │   │ L3 市场      │
│ L4 画布 UI (原型)  │   │             │   │          │   │ L5 团队版    │
│                   │   │             │   │          │   │ L6 运维      │
└──────────────────┘   └──────────────┘   └──────────┘   └──────────────┘
         ▲                    ▲               ▲               ▲
      核心可用            能力完整        体验完善       全功能生产就绪
```

- **当前阶段（MVP）**：聚焦 L0 + L1 + L4，已打通 6 阶段核心流水线（源编排→编译→DAG 并行执行→结果汇聚），实现端到端的工作流自动生成和执行
- **v0.2**：L1 核心引擎功能完整（Compiler 支持完整模板语法、AgentRunner 扩展更多 Provider、并行执行优化）
- **v0.5**：L4 画布 UI 体验完善（可视化编辑、结果详细展示、配置面板增强）
- **v1.0**：L2~L6 全面就绪，进入生产可用阶段
