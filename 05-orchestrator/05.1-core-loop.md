# 05.1 — Orchestrator 核心循环：DAG 并行执行引擎

## 1. 概述

核心循环（Core Loop）是 Orchestrator 的心脏。它接收前端请求，经过 Compiler 编译、DAG 拓扑排序、并行执行、结果汇聚四个阶段，最终返回执行结果。

整个流程在 `agentflow-backend.py` 的 `handle_execute()` 方法中实现。

```
┌─────────────────────────────────────────────────────────┐
│  handle_execute()                                        │
│                                                           │
│  ①  WorkflowJSON 构建 + 校验                              │
│      ├─ nodes[] + edges[] + requirement                    │
│      └─ validate_workflow()                                │
│                                                           │
│  ②  PromptCompiler.compile() → PromptTask[]               │
│      ├─ Kahn 拓扑排序 (topological_sort)                    │
│      ├─ 加载 YAML 模板 (TemplateEngine)                    │
│      ├─ 变量渲染 (render)                                  │
│      └─ 组装 PromptTask[]                                  │
│                                                           │
│  ③  parallel_groups() → 按深度分组                        │
│      ├─ Group 0: [所有 depth=0 的节点]                      │
│      ├─ Group 1: [所有 depth=1 的节点]                      │
│      └─ Group N: [所有 depth=N 的节点]                      │
│                                                           │
│  ④  ThreadPoolExecutor 逐组并行执行                        │
│      ├─ 同组节点：线程池并行 → AgentRunner.execute()        │
│      ├─ 组间节点：串行等待（前一组完成后才启动下一组）        │
│      └─ 每个节点：Think → Act(工具调用) → Observe 循环      │
│                                                           │
│  ⑤  结果汇聚                                              │
│      ├─ 更新 node.result / node.status / node.cost         │
│      ├─ all_results.append(node.to_dict())                 │
│      └─ 清理临时工作目录                                   │
│                                                           │
│  ⑥  返回 JSON                                             │
│      ├─ nodes: [已更新的节点列表]                           │
│      ├─ total_cost: 所有节点成本总和                        │
│      ├─ total_duration: 所有节点耗时总和                    │
│      └─ groups: [每层节点数]                                │
└─────────────────────────────────────────────────────────┘
```

## 2. 步骤详解

### 步骤 ①：WorkflowJSON 构建与校验

**输入：** HTTP POST 请求体 `{nodes, edges, requirement}`

```python
def handle_execute(self):
    body = json.loads(self.rfile.read(length))
    nodes = body.get("nodes", [])
    edges = body.get("edges", [])
    requirement = body.get("requirement", "")

    # 构建 WorkflowJSON
    wf = WorkflowJSON.from_api_request(nodes, requirement, edges)
    errs = validate_workflow(wf)
```

**校验规则（`validate_workflow`）：**
- `nodes` 非空
- 每条 edge 的 `source`/`target` 必须存在于 `nodes`
- 不允许自环（`source == target`）
- 每个节点的 `profile` 必须是 `analysis/design/dev/test/doc/deploy` 之一

### 步骤 ②：PromptCompiler.compile()

```python
from prompt_compiler import PromptCompiler

compiler = PromptCompiler(TEMPLATE_DIR)  # TEMPLATE_DIR = "templates/"
tasks = compiler.compile(wf)
print(f"Compiled {len(tasks)} PromptTasks for {len(nodes)} nodes")
```

`PromptCompiler.compile()` 内部执行：
1. `topological_sort(wf.nodes, wf.edges)` — Kahn 算法排序
2. 对每个节点：`TemplateEngine.load(profile)` → 加载 YAML 模板
3. `TemplateEngine.render(tmpl, variables)` → 渲染提示词
4. `_build_upstream_context()` → 构建上游上下文
5. 组装 `PromptTask(task_id, node_id, prompt, system_prompt, tool_set, max_turns, timeout_s, ...)`

详见 [04.1-compiler-prompt.md](../04-prompt-compiler-agent/04.1-compiler-prompt.md)。

### 步骤 ③：parallel_groups() — DAG 并行分组

```python
from agentflow_schema import parallel_groups

groups = parallel_groups(wf.nodes, wf.edges)
print(f"DAG groups: {[len(g) for g in groups]}")
```

`parallel_groups()` 基于 Kahn 拓扑排序的结果，按深度（depth）将节点分组：

```
输入 DAG:
  analysis ──→ dev ──→ test ──→ doc
       │                      ▲
       └──────────────────────┘

拓扑深度:
  analysis: depth=0  (无依赖)
  dev:      depth=1  (依赖 analysis)
  test:     depth=2  (依赖 dev)
  doc:      depth=2  (依赖 analysis + test)

输出分组:
  Group 0: [analysis]          ← 串行
  Group 1: [dev]               ← 串行
  Group 2: [test, doc]         ← 并行！（同深度，无相互依赖）
```

**关键规则：**
- 同深度节点之间无依赖关系，可以安全并行
- 组间严格串行：Group N 全部完成后才启动 Group N+1
- 空组自动跳过

### 步骤 ④：ThreadPoolExecutor 并行执行

```python
work_dir = tempfile.mkdtemp(prefix="agentflow_")
all_results = []

for group_idx, group_nodes in enumerate(groups):
    # 同组节点：线程池并行
    with ThreadPoolExecutor(max_workers=len(group_nodes)) as pool:
        fut_to_node = {}
        for node in group_nodes:
            task = next((t for t in tasks if t.node_id == node.id), None)
            if not task:
                continue
            node_dir = os.path.join(work_dir, f"node_{node.id}")
            os.makedirs(node_dir, exist_ok=True)
            fut = pool.submit(self._execute_one_node, node, task, node_dir)
            fut_to_node[fut] = node

        # 收集并行执行结果
        for fut in as_completed(fut_to_node):
            node = fut_to_node[fut]
            try:
                result = fut.result()
            except Exception as e:
                result = {
                    "status": "error",
                    "result": f"执行异常: {e}",
                    "cost": 0, "duration": 0, "turns": 0,
                }
            # 更新节点状态
            node.result = result.get("result", "")
            node.output = result.get("output", "")[:2000]
            node.status = result.get("status", "error")
            node.cost = result.get("cost", 0)
            node.duration = result.get("duration", 0)
            node.turns = result.get("turns", 0)
            ...
            all_results.append(node.to_dict())
```

**执行模式：**

| 组大小 | 执行方式 | 说明 |
|--------|---------|------|
| 1 个节点 | 单线程 | `ThreadPoolExecutor(max_workers=1)` 直接执行 |
| N 个节点 | 并行 | `ThreadPoolExecutor(max_workers=N)` 并发执行 |
| 空组 | 跳过 | depth 空洞自动忽略 |

### `_execute_one_node()` — 单个节点执行

```python
def _execute_one_node(self, node, task, node_dir):
    agent_model = node.model or DEFAULT_AGENT_MODEL
    agent = AgentRunner(model=agent_model)

    prompt = task.prompt  # Compiler 生成的完整提示词
    result = agent.execute(
        prompt=prompt,
        work_dir=node_dir,
        profile=node.profile,
        tools_enabled=True,
        max_turns=task.max_turns,
        timeout=task.timeout_s,
    )

    return {
        "result": result.get("result", "完成"),
        "output": result.get("output", "")[:2000],
        "cost": result.get("cost", 0),
        "duration": result.get("duration_ms", 0),
        "status": result.get("status", "ok"),
        "turns": result.get("turns", 1),
        "model": result.get("model", agent_model),
        "provider": result.get("provider", ""),
    }
```

**AgentRunner 执行循环：**

```
AgentRunner.execute()
    │
    ▼
┌──────────────────────────────────────┐
│  Agent Loop                           │
│                                        │
│  ┌──────┐    ┌──────┐    ┌─────────┐  │
│  │ Think│───►│ Act  │───►│ Observe │  │
│  │(LLM) │    │(tool)│    │(result) │  │
│  └──────┘    └──────┘    └─────────┘  │
│       │         │            │         │
│       └─────────┴────────────┘         │
│               │                        │
│          max_turns 耗尽?               │
│          或 timeout 超时?               │
│               │                        │
│         ┌─────┴─────┐                 │
│         │ 是         │ 否              │
│         ▼           ▼                 │
│     返回结果     继续循环              │
└──────────────────────────────────────┘
```

可用工具（从模板的 `tool_set` 加载）：
- **bash** — 执行 shell 命令
- **read** — 读取文件
- **write** — 写入文件

### 步骤 ⑤：结果汇聚

每个节点执行完成后，Orchestrator 更新该节点的状态字段：

```python
node.result   = result.get("result", "")       # 执行结果摘要
node.output   = result.get("output", "")[:2000] # 详细输出（截断前2000字符）
node.status   = result.get("status", "error")    # ok/error/timeout
node.cost     = result.get("cost", 0)            # API 调用成本（美元）
node.duration = result.get("duration", 0)        # 执行耗时（毫秒）
node.turns    = result.get("turns", 0)           # Think→Act 轮数
node.provider = result.get("provider", "")        # LLM 提供商
node.model    = result.get("model", agent_model)  # 使用的模型
```

### 步骤 ⑥：返回 JSON

```python
self.send_response(200)
self.wfile.write(json.dumps({
    "nodes": all_results,                    # 更新后的节点列表
    "total_cost": sum(r.get("cost", 0) for r in all_results),      # 总成本
    "total_duration": sum(r.get("duration", 0) for r in all_results),# 总耗时
    "groups": [len(g) for g in groups],      # 每层节点数

```

## 3. 完整流程伪代码

```python
def handle_execute(self):
    # ① 解析请求
    nodes, edges, requirement = parse_request(body)

    # ② 构建 + 校验 WorkflowJSON
    wf = WorkflowJSON.from_api_request(nodes, requirement, edges)
    validate_workflow(wf)

    # ③ Compiler 编译
    compiler = PromptCompiler(TEMPLATE_DIR)
    tasks = compiler.compile(wf)

    # ④ DAG 分组
    groups = parallel_groups(wf.nodes, wf.edges)

    # ⑤ 逐组并行执行
    work_dir = tempfile.mkdtemp(prefix="agentflow_")
    all_results = []

    for group_nodes in groups:
        with ThreadPoolExecutor(max_workers=len(group_nodes)) as pool:
            futures = {pool.submit(execute_one, node, tasks, work_dir): node
                       for node in group_nodes}

            for fut in as_completed(futures):
                node = futures[fut]
                update_node_from_result(node, fut.result())
                all_results.append(node.to_dict())

    # ⑥ 清理
    shutil.rmtree(work_dir, ignore_errors=True)

    # ⑦ 返回
    return json_response(all_results, total_cost, total_duration)
```

## 4. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **执行引擎** | `ThreadPoolExecutor` | Python 内置，GIL 对 LLM API 调用无影响（IO 密集型） |
| **执行隔离** | 进程内直接调用 | 无 Docker 开销，延迟更低 |
| **并行粒度** | 拓扑深度分组 | 保证正确性的前提下最大化并行度 |
| **错误处理** | 单节点异常不影响同组 | 节点独立执行，互不依赖 |
| **结果存储** | 内存中更新节点对象 | 简单高效，数据量小 |
| **工作目录** | `tempfile.mkdtemp` + 清理 | 避免临时文件残留 |

## 5. 配置参考

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | 9600 | HTTP 服务端口 |
| `DEFAULT_AGENT_MODEL` | `glm-5-turbo` | 默认 LLM 模型（环境变量 `AGENT_MODEL`） |
| `TEMPLATE_DIR` | `templates/` | YAML 模板目录 |
| `max_workers` | `len(group_nodes)` | 每组的最大线程数（动态） |

## 6. 注意事项

1. **幂等性**：每个节点应设计为可重入执行，重试不产生副效应
2. **资源回收**：无论成功或失败，`work_dir` 必须清理（`shutil.rmtree`）
3. **结果截断**：`node.output` 截断到前 2000 字符，避免响应体过大
4. **错误隔离**：`fut.result()` 抛异常时，只影响当前节点，不影响同组其他节点
5. **无 Docker**：所有 Agent 在进程中直接运行，无需容器运行时
